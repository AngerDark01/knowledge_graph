# 问题3: 节点移动后位置没有保存

### 问题描述
拖拽移动节点或群组后，位置信息没有被保存到store中。

### 原因分析
1. ReactFlow在拖拽过程中发送position事件，拖拽结束时只发送dragging: false事件
2. 拖拽结束事件不包含具体的位置信息
3. 原来的处理逻辑只处理包含position信息的事件

### 解决方案
1. 检测dragging: false事件
2. 在事件中没有位置信息时，通过reactFlowInstance.getNode()获取节点的当前最终位置
3. 将获取到的位置保存到store中

### 关键代码模式
```typescript
// 拖拽结束时（dragging: false），获取节点当前位置并保存
if ('dragging' in change && change.dragging === false) {
  // 获取节点当前最终位置
  const currentNode = reactFlowInstance?.getNode(change.id);
  
  if (currentNode) {
    // 更新到store
    updateNode(change.id, {
      position: currentNode.position  // 保存节点的新位置
    });
  }
}
```

## 三个金律 🎯

### 1. **单一职责原则**
- **节点组件**：只负责渲染，不处理状态更新
- **ReactFlow**：负责 UI 交互和事件触发
- **onNodesChange**：唯一的状态同步桥梁
- **Store**：只负责数据存储

### 2. **明确保留原则**
更新节点时，**必须明确保留所有未修改的属性**：
```typescript
updateNode(id, {
  position: currentNode.position,  // 明确写出！
  width: newWidth,
  height: newHeight,
});
```

### 3. **防抖延迟原则**
- 只在 `resizing = false` 或 `dragging = false` 时更新 store
- 使用 100-200ms 防抖避免频繁更新
- 先应用到 ReactFlow（UI 立即响应），再异步更新 store

## 一句话记忆法

**"节点组件不碰状态，onNodesChange 防抖处理，更新必须保留位置，store 深度合并属性"**

把这份文档保存下来，下次遇到类似的可调整大小、可拖拽的复杂交互场景时，按照这个模式来设计，就能避免 90% 的状态同步问题！
