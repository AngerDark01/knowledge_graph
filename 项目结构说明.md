# 知识图谱项目结构说明

## 项目概述
这是一个基于 Next.js 和 React Flow 的知识图谱编辑器，支持创建、编辑、连接节点，以及对节点进行分组管理。

## 文件夹结构
```
src/
├── app/                 # 应用入口和页面结构
├── components/          # UI 组件
│   ├── graph/          # 图编辑器相关组件
│   └── ui/             # 通用 UI 组件
├── stores/             # 状态管理
│   └── graph/          # 图数据状态管理
├── types/              # TypeScript 类型定义
│   └── graph/          # 图数据类型定义
└── utils/              # 工具函数
    └── graph/          # 图相关工具函数
```

## 详细模块说明

### 1. src/app/
- `page.tsx`：应用主页面，集成 GraphPage 组件
- `layout.tsx`：应用根布局
- `globals.css`：全局样式
- `favicon.ico`：网站图标

### 2. src/components/graph/
- `GraphPage.tsx`：图编辑器主页面
  - 实现了完整的图编辑器界面，包括节点画布和属性编辑面板
  - 集成 React Flow API，处理节点拖拽、连接、删除等操作
  - 实现了键盘快捷键和画布控制功能
  - 处理节点/群组的创建、更新、删除操作
  - 包含群组内节点的边界约束逻辑
  - 实现了尺寸调整的防抖机制
- `CustomNode.tsx`：自定义普通节点组件
  - 继承 BaseNode，设置为普通节点
  - 支持尺寸调整
  - 显示节点内容
- `CustomGroup.tsx`：自定义群组节点组件
  - 继承 BaseNode，设置为群组节点
  - 处理群组标题编辑
  - 显示群组中节点的数量
- `BaseNode.tsx`：节点基类组件
  - 提供统一的节点结构和样式
  - 实现上下左右四个方向的连接点
  - 支持标题双击编辑功能
  - 添加尺寸调整控制功能
- `CustomEdge.tsx`：自定义边组件
  - 实现自定义样式的连接线
  - 支持边标签显示
  - 支持自定义颜色和线宽
- `NodeEditor.tsx`：节点属性编辑器
  - 提供节点属性编辑界面
  - 支持标题、内容编辑
  - 支持节点分组管理
- `EdgeEditor.tsx`：边属性编辑器
  - 提供边属性编辑界面
  - 支持标签、颜色、线宽编辑

### 3. src/components/ui/
- `button.tsx`：通用按钮组件
- `card.tsx`：卡片组件
- `input.tsx`：输入框组件
- `label.tsx`：标签组件
- `select.tsx`：选择框组件
- `textarea.tsx`：文本域组件
- （这些是 shadcn/ui 库的组件实现）

### 4. src/stores/graph/
- `index.ts`：状态管理主入口
  - 合并 nodesSlice、edgesSlice 和 canvasViewSlice
  - 导出 useGraphStore hook
- `nodesSlice.ts`：节点状态管理
  - 管理节点和群组的 CRUD 操作
  - 处理节点与群组的关系管理
  - 实现群组边界的自动计算功能
  - 包含节点验证逻辑
- `edgesSlice.ts`：边状态管理
  - 管理边的 CRUD 操作
  - 处理边属性更新
- `canvasViewSlice.ts`：画布视图状态管理
  - 管理视口位置和缩放状态
  - 管理画布尺寸

### 5. src/types/graph/
- `models.ts`：图数据模型类型定义
  - `BlockEnum`：节点类型枚举（节点、群组）
  - `Node`：普通节点类型定义
  - `Group`：群组节点类型定义
  - `Edge`：边类型定义
  - Zod 验证 schema：用于数据验证

### 6. src/utils/graph/
- `test-utils.ts`：测试工具函数
  - 提供节点和边操作的测试用例
  - 包含 CRUD 操作的测试函数

## 关键功能模块

### 1. 节点系统
- 普通节点和群组节点的实现
- 节点尺寸调整功能
- 节点连接点（上下左右四个方向）
- 节点分组管理

### 2. 状态管理
- 使用 Zustand 实现集中状态管理
- 分离节点、边和画布视图状态
- 支持数据持久化（通过状态管理）

### 3. 用户交互
- 拖拽创建和移动节点
- 连接节点（支持防重复连接）
- 键盘快捷键支持
- 双击编辑节点标题

### 4. UI/UX
- 响应式布局
- 节点选择和高亮
- 缩放和平移控制
- 属性编辑面板

### 5. 数据模型
- 定义清晰的节点、群组和边数据结构
- 支持丰富的节点属性
- 包含创建和更新时间戳

## 项目架构特点

1. **模块化设计**：功能按组件分离，便于维护
2. **状态管理**：使用 Zustand 集中管理状态
3. **类型安全**：使用 TypeScript 和 Zod 进行类型验证
4. **可扩展性**：组件设计支持自定义扩展
5. **性能优化**：使用防抖等技术优化用户体验
