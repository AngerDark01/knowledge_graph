# 知识图谱项目开发经验总结

## 主要错误类型及解决方案

### 1. 模块缺失错误
- **错误**: `Cannot find module '@/components/ui/input'`
- **原因**: 使用了未安装的 shadcn/ui 组件
- **解决方案**: 通过 `npx shadcn@latest add [组件名]` 命令添加缺失组件
- **规避方法**: 在使用新的 shadcn/ui 组件前，先确认该组件已通过 CLI 安装

### 2. CSS 配置错误
- **错误**: `Can't resolve 'tw-animate-css'`
- **原因**: 不存在的 CSS 插件被引入
- **解决方案**: 从 `globals.css` 中移除不存在的导入语句
- **规避方法**: 仔细检查导入的模块，确保其存在且正确安装

### 3. ReactFlow API 使用错误
- **错误**: `setReactFlowInstance is not a function`, `flowInstance.on is not a function`
- **原因**: 对 ReactFlow API 的参数和方法理解不准确
- **解决方案**: 
  - 正确理解 `onInit` 回调的参数用法
  - 确保在组件挂载后再使用 ReactFlow 实例的方法
- **规避方法**: 仔细阅读官方文档，理解 API 的正确用法，使用类型检查确保正确使用

### 4. 类型错误
#### 隐式 any 类型
- **错误**: `Parameter 'node' implicitly has an 'any' type`
- **原因**: TypeScript 无法推断回调函数参数类型
- **解决方案**: 为函数参数显式添加类型注解
- **规避方法**: 在使用数组方法（如 filter、forEach）时，始终为回调函数参数添加类型

#### 类型不兼容
- **错误**: `Type '"loose"' is not assignable to type 'ConnectionMode | undefined'`
- **原因**: 使用了错误的类型值
- **解决方案**: 导入正确的类型并使用枚举值
- **规避方法**: 使用导入的类型常量而非字符串字面量

### 5. JSX 元素属性错误
- **错误**: `JSX elements cannot have multiple attributes with the same name`
- **原因**: 在 JSX 元素上重复定义了相同属性
- **解决方案**: 移除重复的属性定义
- **规避方法**: 在代码编辑时仔细检查，避免重复添加相同属性

### 6. 节点类型不匹配
- **错误**: `Type '{ custom: React.NamedExoticComponent<...>; }' is not assignable to type 'NodeTypes'`
- **原因**: 自定义节点组件的类型签名与 ReactFlow 要求不匹配
- **解决方案**: 使用 NodeProps 作为组件的泛型类型参数
- **规避方法**: 在创建自定义节点组件时，使用正确的类型签名

### 7. 重复的属性定义
- **错误**: `JSX elements cannot have multiple attributes with the same name`
- **原因**: 在 JSX 元素上重复定义了相同的属性（如 className）
- **解决方案**: 合并重复的属性到单个属性中
- **规避方法**: 在编写 JSX 时仔细检查，避免重复添加相同属性

### 8. 类型断言和类型注解
- **错误**: `Parameter 'edge' implicitly has an 'any' type`
- **原因**: TypeScript 无法准确推断某些变量的类型
- **解决方案**: 使用适当的类型注解或类型断言
- **规避方法**: 确保接口定义完整，使用类型守卫和断言明确类型信息

### 9. ReactFlow 事件处理
- **错误**: `Property 'on' does not exist on type 'ReactFlowInstance'`
- **原因**: TypeScript 定义中未包含特定方法
- **解决方案**: 使用类型守卫（`in` 操作符）安全地访问方法
- **规避方法**: 在调用可能不存在的方法前，先检查其是否存在

## 最佳实践

### 1. 环境配置
- 确保 Node.js 版本满足项目要求
- 使用 pnpm 作为包管理器
- 按需安装 shadcn/ui 组件

### 2. 类型安全
- 始终为函数参数添加类型注解
- 使用 TypeScript 接口定义数据结构
- 在使用第三方库时，导入正确的类型定义

### 3. ReactFlow 使用
- 正确理解 ReactFlow 实例的方法和事件
- 在组件挂载后再使用 ReactFlow 实例
- 使用正确的类型常量而非字符串字面量

### 4. 代码组织
- 按功能模块化组织代码
- 使用 Zustand 的切片模式管理状态
- 合理分离 UI 组件和业务逻辑

## 有用的调试技巧

1. 遇到模块缺失错误时，使用 `npx shadcn@latest add [组件名]` 添加组件
2. 遇到 API 使用错误时，查阅官方文档确认正确的使用方式
3. 遇到类型错误时，为参数显式添加类型注解
4. 遇到运行时错误时，检查组件生命周期，确保在正确的时间访问实例方法
