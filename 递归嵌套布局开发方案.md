# 递归嵌套布局系统 - 详细开发方案

## 一、项目现状分析

### 1.1 当前实现
- ✅ **顶层布局算法**：在画布顶层使用网格布局（2列）
- ✅ **配置常量**：`GROUP_PADDING`、`NODE_VISUAL_PADDING` 等配置
- ✅ **边界管理**：自动扩展群组边界以包含所有子节点
- ✅ **位置约束**：节点在群组内的位置约束
- ✅ **单层嵌套**：支持 Group 包含 Node（但不支持 Group 包含 Group）

### 1.2 当前限制
- ❌ 布局算法与节点创建逻辑耦合在 `useNodeHandling.ts` 中
- ❌ 不支持 Group 嵌套 Group
- ❌ 没有独立的布局引擎，难以复用
- ❌ 缺少针对已有子节点的重新布局功能
- ❌ 布局参数硬编码，不够灵活

### 1.3 关键配置
```typescript
// src/stores/graph/nodes/types.ts
GROUP_PADDING = { top: 70, left: 20, right: 20, bottom: 20 }
NODE_VISUAL_PADDING = 4

// 节点默认尺寸
NODE: { width: 350, height: 280 }  // 收缩态
GROUP: { width: 300, height: 200 }

// 网格布局参数（当前在 useNodeHandling.ts 中）
nodesPerRow: 2
nodeSpacingX: 380  // 节点宽度(350) + 间距(30)
nodeSpacingY: 310  // 节点高度(280) + 间距(30)
```

## 二、设计目标

### 2.1 核心目标
1. **分离关注点**：布局算法独立于节点创建逻辑
2. **递归支持**：支持任意层级的 Group 嵌套
3. **灵活配置**：开发者可调整布局参数（配置文件）
4. **增量布局**：选中容器节点，只布局其直接子节点
5. **递归布局**：提供全局递归布局功能
6. **保持兼容**：不破坏现有功能和数据结构

### 2.2 用户交互流程
```
情景1: 增量布局（针对选中节点的直接子节点）
用户选中节点 → 点击"布局子节点"按钮 → 只布局直接子节点 → 更新边界

情景2: 递归布局（全局）
用户点击"递归布局"按钮 → 递归布局所有嵌套结构 → 更新所有边界

情景3: 自动布局（创建时）
用户在Group内添加节点 → 自动应用布局算法 → 避免重叠
```

### 2.3 技术约束
- 不破坏现有数据模型（Node、Group、Edge）
- 不影响现有的拖拽、编辑、历史记录等功能
- 配置文件对开发者可见可调，对用户不可见
- 父节点收缩（collapsed=true）时，不隐藏子节点，只是UI表现不同
- 父节点最小尺寸受子节点位置影响（不能遮盖子节点）

#### 3.2.3 网格布局策略 (`src/utils/graph/layout/strategies/GridLayoutStrategy.ts`)
```typescript
export class GridLayoutStrategy implements LayoutStrategy {
  name = 'grid';

  calculateLayout(
    container: Group,
    children: (Node | Group)[],
    config: LayoutConfig
  ): Map<string, { x: number; y: number }> {
    const positions = new Map<string, { x: number; y: number }>();
    const { columns, horizontalSpacing, verticalSpacing } = config.grid;
    const padding = config.containerPadding;
    const containerPos = safePosition(container.position);

    children.forEach((child, index) => {
      const row = Math.floor(index / columns);
      const col = index % columns;

      const nodeSize = this.getNodeSize(child, config);
      const totalNodeWidth = nodeSize.width + horizontalSpacing;
      const totalNodeHeight = nodeSize.height + verticalSpacing;

      // 计算绝对位置
      const x = containerPos.x + padding.left + (col * totalNodeWidth);
      const y = containerPos.y + padding.top + (row * totalNodeHeight);

      positions.set(child.id, { x, y });
    });

    return positions;
  }

  private getNodeSize(node: Node | Group, config: LayoutConfig): { width: number; height: number } {
    // 获取节点实际尺寸或默认尺寸
    return {
      width: node.width || config.defaultNodeSize[node.type].width,
      height: node.height || config.defaultNodeSize[node.type].height,
    };
  }
}
```

#### 3.2.4 布局引擎 (`src/utils/graph/layout/LayoutEngine.ts`)
```typescript
export class LayoutEngine {
  private strategy: LayoutStrategy;
  private config: LayoutConfig;

  constructor(strategy: LayoutStrategy, config: LayoutConfig = defaultLayoutConfig) {
    this.strategy = strategy;
    this.config = config;
  }

  /**
   * 布局容器的直接子节点
   */
  layoutChildren(
    container: Group,
    children: (Node | Group)[]
  ): Map<string, { x: number; y: number }> {
    return this.strategy.calculateLayout(container, children, this.config);
  }

  /**
   * 递归布局容器及其所有后代
   */
  layoutRecursive(
    container: Group,
    allNodes: (Node | Group)[]
  ): Map<string, { x: number; y: number }> {
    const allPositions = new Map<string, { x: number; y: number }>();

    // 递归布局函数
    const layoutNode = (parentContainer: Group) => {
      // 获取直接子节点
      const directChildren = allNodes.filter(node => {
        if (node.type === BlockEnum.NODE) {
          return (node as Node).groupId === parentContainer.id;
        } else {
          return node.parentId === parentContainer.id;
        }
      });

      if (directChildren.length === 0) return;

      // 布局直接子节点
      const positions = this.layoutChildren(parentContainer, directChildren);
      positions.forEach((pos, nodeId) => {
        allPositions.set(nodeId, pos);
      });

      // 递归处理子 Group
      directChildren
        .filter(child => child.type === BlockEnum.GROUP)
        .forEach(childGroup => {
          layoutNode(childGroup as Group);
        });
    };

    layoutNode(container);
    return allPositions;
  }

  /**
   * 布局所有顶层节点和群组
   */
  layoutTopLevel(
    nodes: (Node | Group)[],
    canvasCenter: { x: number; y: number }
  ): Map<string, { x: number; y: number }> {
    // 获取所有顶层节点（没有 parent 的节点）
    const topLevelNodes = nodes.filter(node => {
      if (node.type === BlockEnum.NODE) {
        return !(node as Node).groupId;
      } else {
        return !node.parentId;
      }
    });

    // 创建一个虚拟容器用于计算布局
    const virtualContainer: Group = {
      id: 'virtual-root',
      type: BlockEnum.GROUP,
      position: canvasCenter,
      title: 'Virtual Root',
      collapsed: false,
      nodeIds: [],
      boundary: { minX: 0, minY: 0, maxX: 0, maxY: 0 },
      createdAt: new Date(),
      updatedAt: new Date(),
    };

    return this.layoutChildren(virtualContainer, topLevelNodes);
  }
}
```

#### 3.2.5 Store 扩展 (`src/stores/graph/layoutSlice.ts`)
```typescript
export interface LayoutSlice {
  layoutConfig: LayoutConfig;

  /**
   * 布局指定容器的直接子节点
   */
  layoutContainerChildren: (containerId: string) => void;

  /**
   * 递归布局指定容器及其所有后代
   */
  layoutContainerRecursive: (containerId: string) => void;

  /**
   * 布局所有顶层节点
   */
  layoutTopLevel: () => void;

  /**
   * 递归布局整个画布
   */
  layoutAll: () => void;

  /**
   * 更新布局配置（开发者使用）
   */
  updateLayoutConfig: (config: Partial<LayoutConfig>) => void;
}

export const createLayoutSlice = (set: any, get: any): LayoutSlice => {
  const engine = new LayoutEngine(
    new GridLayoutStrategy(),
    defaultLayoutConfig
  );

  return {
    layoutConfig: defaultLayoutConfig,

    layoutContainerChildren: (containerId: string) => {
      const state = get();
      const container = state.nodes.find((n: Node | Group) =>
        n.id === containerId && n.type === BlockEnum.GROUP
      ) as Group;

      if (!container) {
        console.warn('Container not found:', containerId);
        return;
      }

      // 获取直接子节点
      const children = state.nodes.filter((n: Node | Group) => {
        if (n.type === BlockEnum.NODE) {
          return (n as Node).groupId === containerId;
        } else {
          return n.parentId === containerId;
        }
      });

      if (children.length === 0) return;

      // 计算新位置
      const newPositions = engine.layoutChildren(container, children);

      // 更新节点位置
      const updatedNodes = state.nodes.map((node: Node | Group) => {
        if (newPositions.has(node.id)) {
          return {
            ...node,
            position: newPositions.get(node.id)!,
            updatedAt: new Date(),
          };
        }
        return node;
      });

      set({ nodes: updatedNodes });

      // 更新容器边界
      setTimeout(() => {
        get().updateGroupBoundary(containerId);
      }, 50);

      // 添加历史记录
      if (get().addHistorySnapshot) {
        get().addHistorySnapshot();
      }
    },

    layoutContainerRecursive: (containerId: string) => {
      const state = get();
      const container = state.nodes.find((n: Node | Group) =>
        n.id === containerId && n.type === BlockEnum.GROUP
      ) as Group;

      if (!container) {
        console.warn('Container not found:', containerId);
        return;
      }

      // 递归计算所有后代的新位置
      const newPositions = engine.layoutRecursive(container, state.nodes);

      if (newPositions.size === 0) return;

      // 更新所有节点位置
      const updatedNodes = state.nodes.map((node: Node | Group) => {
        if (newPositions.has(node.id)) {
          return {
            ...node,
            position: newPositions.get(node.id)!,
            updatedAt: new Date(),
          };
        }
        return node;
      });

      set({ nodes: updatedNodes });

      // 递归更新所有容器边界（从内到外）
      const updateBoundariesRecursive = (groupId: string) => {
        // 先更新子 Group 的边界
        const childGroups = updatedNodes.filter((n: Node | Group) =>
          n.type === BlockEnum.GROUP && n.parentId === groupId
        );
        childGroups.forEach((childGroup: Group) => {
          updateBoundariesRecursive(childGroup.id);
        });

        // 再更新当前 Group 的边界
        get().updateGroupBoundary(groupId);
      };

      setTimeout(() => {
        updateBoundariesRecursive(containerId);
      }, 50);

      // 添加历史记录
      if (get().addHistorySnapshot) {
        get().addHistorySnapshot();
      }
    },

    layoutTopLevel: () => {
      const state = get();
      const canvasCenter = { x: 400, y: 300 }; // 可以从 viewport 计算

      const newPositions = engine.layoutTopLevel(state.nodes, canvasCenter);

      if (newPositions.size === 0) return;

      const updatedNodes = state.nodes.map((node: Node | Group) => {
        if (newPositions.has(node.id)) {
          return {
            ...node,
            position: newPositions.get(node.id)!,
            updatedAt: new Date(),
          };
        }
        return node;
      });

      set({ nodes: updatedNodes });

      // 更新所有顶层 Group 的边界
      updatedNodes
        .filter((n: Node | Group) =>
          n.type === BlockEnum.GROUP && !n.parentId
        )
        .forEach((group: Group) => {
          setTimeout(() => {
            get().updateGroupBoundary(group.id);
          }, 50);
        });

      // 添加历史记录
      if (get().addHistorySnapshot) {
        get().addHistorySnapshot();
      }
    },

    layoutAll: () => {
      const state = get();

      // 1. 先布局所有顶层节点
      get().layoutTopLevel();

      // 2. 然后递归布局每个顶层 Group
      const topLevelGroups = state.nodes.filter((n: Node | Group) =>
        n.type === BlockEnum.GROUP && !n.parentId
      ) as Group[];

      topLevelGroups.forEach((group) => {
        get().layoutContainerRecursive(group.id);
      });
    },

    updateLayoutConfig: (config: Partial<LayoutConfig>) => {
      set({
        layoutConfig: {
          ...get().layoutConfig,
          ...config,
        }
      });

      // 重新创建布局引擎
      const newEngine = new LayoutEngine(
        new GridLayoutStrategy(),
        get().layoutConfig
      );
      // 注意: 这里需要更新 engine 引用，可能需要调整实现
    },
  };
};
```

## 五、关键技术点

### 5.1 递归布局的关键
```typescript
// 关键：需要从内到外更新边界
// 错误做法：同时更新所有边界 ❌
allGroups.forEach(group => updateBoundary(group));

// 正确做法：深度优先，从叶子到根 ✅
function updateBoundariesDFS(groupId: string) {
  // 1. 先更新所有子 Group
  const childGroups = getChildGroups(groupId);
  childGroups.forEach(child => updateBoundariesDFS(child.id));

  // 2. 再更新当前 Group
  updateBoundary(groupId);
}
```

### 5.2 坐标系统
- **Store 存储**：统一使用绝对坐标（相对于画布原点）
- **ReactFlow 渲染**：群组内节点使用相对坐标
- **布局计算**：统一在绝对坐标系中计算

### 5.3 性能优化
```typescript
// 批量更新节点位置

  if (newPositions.has(node.id)) {
    return { ...node, position: newPositions.get(node.id)! };

set({ nodes: updatedNodes }); // 一次性更新

// 延迟更新边界，避免阻塞 UI

}, 50);
```

## 七、注意事项

### 7.1 不破坏现有功能
- ✅ 不修改现有的数据模型
- ✅ 不影响现有的拖拽、编辑功能
- ✅ 布局是可选的，用户可以手动调整
- ✅ 历史记录正常工作

### 7.2 关于 collapsed 属性
根据用户需求：
- `collapsed=true` **不隐藏**子节点
- `collapsed` 只影响 UI 表现（例如，标题栏显示"已收缩"标识）
- 父节点的最小尺寸始终受子节点位置影响（不能遮盖子节点）

### 7.3 关于自动布局
用户明确提到："自动布局目前不需要"
- 不在节点创建时自动触发布局
- 保留现有的网格布局逻辑（在 `useNodeHandling` 中）
- 布局功能通过按钮**手动触发**

### 7.4 边界管理
- 边界管理机制已经实现，保持不变
- 布局完成后需要调用 `updateGroupBoundary`
- 注意递归更新边界的顺序（从内到外）
