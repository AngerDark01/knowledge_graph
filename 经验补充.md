# ReactFlow 节点展开/收缩功能实现经验 - 补充

## 问题描述

在实现 NoteNode 的展开/收缩功能时，遇到 UI 更新延迟的问题：

点击展开/收缩按钮后，节点尺寸更新不立即反映在 UI 上，需要点击其他节点或进行其他操作才能看到变化。

## 问题根本原因

ReactFlow 内部有拖拽处理机制，当在节点内部进行某些操作时，可能误触发 `onNodeDragStart` 事件。
一旦触发拖拽，`isDraggingRef.current` 被设置为 `true`。
GraphPageContent 中的节点同步逻辑会检查 `isDraggingRef.current`，如果为 `true` 则跳过同步以避免覆盖用户拖拽操作。
因此，即使 store 中的节点尺寸已更新，但由于同步被跳过，UI 没有立即更新。

## 解决方案

为 NoteNode 中的交互元素添加 `nodrag` 类名和事件冒泡阻止：

1. 在展开/收缩按钮上添加 `nodrag` 类名
2. 添加 `onMouseDown`、`onMouseUp`、`onMouseMove` 事件处理器来阻止事件冒泡
3. 在 NoteNode 中添加依赖于 store 状态变化的 useEffect 来确保状态同步

## 关键知识点

1. **ReactFlow 的 nodrag 机制**：带有 `nodrag` 类的元素不会触发节点拖拽
2. **事件冒泡控制**：使用 `stopPropagation()` 阻止事件冒泡可以避免意外触发父级事件处理
3. **状态同步**：确保组件内部状态与全局 store 状态保持同步的重要性
