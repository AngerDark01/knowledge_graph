# 核心架构

## 项目概述

知识图谱编辑器是一个基于 React 和 ReactFlow 构建的交互式图形编辑应用，用于创建、编辑和管理知识图谱。项目支持节点的创建、分组、关联，以及丰富的交互功能如拖拽、缩放、历史记录等。

### 核心特性
- **节点管理**: 创建、编辑、删除普通节点和群组节点
- **群组功能**: 节点分组管理，支持群组内节点的位置约束
- **关系管理**: 节点间的连接关系，支持跨群组关系
- **交互功能**: 拖拽、缩放、展开/收缩、历史撤销/重做
- **数据验证**: 使用 Zod 进行严格的数据验证
- **Markdown 支持**: 节点内容支持 Markdown 格式
- **自动布局**: 内置多种布局算法，支持节点自动排列，支持多层嵌套结构
- **智能布局系统**: 采用分层架构，支持画布布局、群组布局、递归布局

### 架构设计原则

1. **分层架构**
   - 表现层（Components）
   - 逻辑层（Hooks & Utils）
   - 数据层（Stores & Types）
   - 服务层（Services）

2. **模块化设计**
   - 每个功能模块独立封装
   - 清晰的职责划分
   - 易于维护和扩展

3. **类型安全**
   - 全面的 TypeScript 类型定义
   - Zod 运行时验证
   - 类型推导和约束

4. **设计模式应用**
   - 策略模式：布局算法的可插拔设计
   - 依赖注入：各层之间的松耦合
   - 单一职责：每个模块只负责一个功能

## 服务层架构 (src/services)

### 布局系统 (src/services/layout)

布局系统采用三层架构设计：

#### 1. 策略层 (strategies/)
- **CanvasLayoutStrategy**: 负责顶层节点的居中布局，使用权重排序
- **GroupLayoutStrategy**: 负责群组内节点的网格布局，保持相对位置
- **RecursiveLayoutStrategy**: 负责递归布局所有嵌套层级，从深到浅

#### 2. 算法层 (algorithms/)
- **GridAlgorithm**: 从左上角开始的网格布局（用于群组内部）
- **GridCenterAlgorithm**: 居中网格布局（用于顶层节点）
- **EdgeOptimizer**: 边连接点优化算法

#### 3. 工具层 (utils/)
- **CoordinateTransformer**: 坐标系统转换
- **GroupSizeAdjuster**: 群组尺寸自动调整
- **CollisionResolver**: 碰撞检测与解决
- **NestingTreeBuilder**: 嵌套层级关系分析
- **GeometryUtils**: 几何计算工具

#### 4. 管理层 (LayoutManager.ts)
- **智能策略选择**: 根据用户选项自动选择最合适的布局策略
- **生命周期管理**: 处理布局操作的开始、取消和完成
- **性能监控**: 记录布局操作的执行时间、迭代次数和碰撞数
