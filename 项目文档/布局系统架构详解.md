# 知识图谱布局系统架构详解

## 1. 整体架构设计

知识图谱布局系统采用分层架构模式，遵循SOLID原则中的依赖倒置原则。整个系统分为四层：

1. **策略层（Strategies）** - 定义不同布局策略的执行逻辑
2. **算法层（Algorithms）** - 实现具体的布局计算算法
3. **工具层（Utils）** - 提供通用的工具函数
4. **管理器层（LayoutManager）** - 协调各层并提供统一接口

## 2. 核心组件详解

### 2.1 LayoutManager（核心管理器）

LayoutManager是布局系统的入口，负责：

- **策略注册**：自动注册默认布局策略
- **智能选择**：根据参数自动选择合适的布局策略
- **生命周期管理**：处理布局操作的开始、取消和完成
- **性能监控**：记录布局操作的执行时间、迭代次数和碰撞数

**依赖关系：**
- 依赖策略层：GroupLayoutStrategy、CanvasLayoutStrategy、RecursiveLayoutStrategy
- 依赖算法层：GridAlgorithm、GridCenterAlgorithm
- 依赖工具层：CollisionResolver、NestingTreeBuilder、EdgeOptimizer

### 2.2 布局策略层

#### 2.2.1 GroupLayoutStrategy（群组布局策略）
- **职责**：对指定群组的直接子节点进行布局
- **特点**：使用固定锚点（群组左上角 + padding），自动调整群组大小
- **流程**：
  1. 筛选目标群组的直接子节点（去重）
  2. 使用算法计算相对布局（从左上角开始）
  3. 转换为绝对坐标（基于群组锚点）
  4. 碰撞检测与解决
  5. 递归更新嵌套子节点位置
  6. 自适应调整群组大小
  7. 返回布局结果

#### 2.2.2 CanvasLayoutStrategy（画布布局策略）
- **职责**：对所有顶层节点（没有groupId）进行布局
- **特点**：使用居中布局算法（以原点为中心），处理嵌套节点位置更新
- **流程**：
  1. 筛选顶层节点
  2. 使用算法计算居中布局
  3. 碰撞检测与解决
  4. 更新嵌套节点位置（保持相对位置）
  5. 优化边的连接点
  6. 返回布局结果

#### 2.2.3 RecursiveLayoutStrategy（递归布局策略）
- **职责**：对所有嵌套层级的节点进行递归布局
- **特点**：从最深层群组开始，逐层向上布局
- **核心思想**：作为GroupLayoutStrategy和CanvasLayoutStrategy的编排器，不重新实现布局逻辑，而是协调其他策略的执行
- **流程**：
  1. 构建嵌套层级树（使用NestingTreeBuilder）
  2. 从最深层到第1层，依次布局每个群组（调用GroupLayoutStrategy）
  3. 布局顶层节点（调用CanvasLayoutStrategy）
  4. 从深到浅调整所有群组大小
  5. 优化所有边
  6. 返回完整的布局结果

### 2.3 布局算法层

#### 2.3.1 GridAlgorithm（网格布局算法）
- **特点**：从(0, 0)开始布局，所有坐标为正数（第四象限）
- **适用场景**：群组内部布局（相对于群组左上角）
- **输出**：节点左上角坐标

#### 2.3.2 GridCenterAlgorithm（网格中心布局算法）
- **特点**：以原点(0, 0)为中心布局，可能产生负坐标
- **适用场景**：全画布布局（顶层节点）
- **特性**：高权重节点可以排在中心位置

### 2.4 工具层

#### 2.4.1 NestingTreeBuilder（嵌套树构建器）
- **职责**：分析节点关系，构建嵌套层级结构，计算嵌套深度
- **功能**：
  - 递归计算节点的嵌套深度
  - 检测循环嵌套
  - 按深度对节点分组
  - 构建嵌套层级树

#### 2.4.2 GroupSizeAdjuster（群组大小调整器）
- **职责**：根据子节点的实际占用空间自动调整父群组的尺寸
- **特点**：假设节点位置是左上角坐标
- **功能**：
  - 调整单个群组大小
  - 批量调整多个群组大小
  - 计算子节点相对于父群组的边界

#### 2.4.3 NestedNodePositionUpdater（嵌套节点位置更新器）
- **职责**：根据父节点位置变化更新子节点的绝对位置
- **功能**：
  - canvas布局中更新嵌套节点位置
  - 群组布局中更新后代节点位置
  - 递归计算嵌套节点的绝对位置

#### 2.4.4 CollisionResolver（碰撞解决器）
- **职责**：使用迭代式排斥力算法解决节点重叠问题
- **特点**：迭代式排斥力算法
- **功能**：
  - 碰撞检测
  - 碰撞解决
  - 碰撞统计

#### 2.4.5 CoordinateTransformer（坐标转换器）
- **职责**：提供各种坐标系统之间的转换功能
- **功能**：
  - 相对坐标转绝对坐标
  - 节点转换到第四象限
  - 群组锚点计算

#### 2.4.6 EdgeOptimizer（边优化器）
- **职责**：优化边的连接点以提供更好的视觉效果
- **功能**：
  - 优化所有边的连接点
  - 计算单条边的最佳连接点
  - 批量优化（带性能优化）

## 3. 触发逻辑和控制流程

### 3.1 用户界面触发
- **LayoutControl组件**提供三种布局按钮：
  - 应用布局（CanvasLayoutStrategy）
  - 布局群组内部（GroupLayoutStrategy）
  - 递归布局全部（RecursiveLayoutStrategy）

### 3.2 策略选择逻辑
1. 如果显式指定了策略，直接使用
2. 如果是递归布局模式，选择'recursive-layout'
3. 如果指定了目标群组ID（群组内布局），选择'group-layout'
4. 默认使用画布布局（顶层节点布局）

### 3.3 布局执行流程
1. **智能策略选择**：LayoutManager根据参数自动选择策略
2. **布局计算**：策略调用算法层进行布局计算
3. **坐标转换**：工具层进行坐标系统转换
4. **碰撞解决**：解决节点之间的重叠
5. **尺寸调整**：调整群组大小
6. **边优化**：优化边的连接点
7. **结果应用**：更新Store中的节点和边数据

## 4. 潜在优化点

### 4.1 代码结构优化

#### 4.1.1 依赖注入模式
当前代码中，依赖关系是硬编码的，可以通过依赖注入模式提高测试性和灵活性：

```typescript
// 当前方式
constructor() {
  this.collisionResolver = new CollisionResolver();
  this.edgeOptimizer = new EdgeOptimizer();
  this.nestingTreeBuilder = new NestingTreeBuilder();
}
```

#### 4.1.2 配置管理优化
配置分散在多个文件中，可以统一管理并提供更好的类型安全。

#### 4.1.3 类型定义优化
当前布局结果中节点位置类型定义不够精确，可以区分绝对坐标和相对坐标类型：

```typescript
// 区分不同坐标系统的类型
type AbsolutePosition = { x: number; y: number };
type RelativePosition = { x: number; y: number };
```

### 4.2 性能优化

#### 4.2.1 缓存机制
为频繁计算的布局结果提供缓存机制，特别是对于大型图谱。

#### 4.2.2 懒加载
对于嵌套层级深的图谱，可以实现懒加载机制，只计算当前视口附近的节点布局。

#### 4.2.3 算法优化
碰撞解决算法可以采用空间分区算法（如四叉树）来优化复杂度。

### 4.3 架构优化

#### 4.3.1 模块拆分
对于大型文件（如LayoutManager.ts），可以考虑按功能进一步拆分：

- LayoutManager.ts - 核心管理逻辑
- LayoutStrategySelector.ts - 策略选择逻辑
- LayoutLifecycleManager.ts - 生命周期管理

#### 4.3.2 事件系统
引入事件系统来解耦各组件之间的通信，比如布局完成事件、进度更新事件等。

#### 4.3.3 策略链模式
对于复杂的布局需求，可以实现策略链模式，允许组合多个布局策略。

## 5. 总结

知识图谱布局系统设计合理，采用了良好的分层架构和设计模式。主要优势包括：

1. **模块化设计**：各层职责清晰，便于维护和扩展
2. **策略模式**：支持多种布局策略，易于添加新策略
3. **工具复用**：工具层实现了通用功能，减少代码重复
4. **智能选择**：自动根据参数选择合适的布局策略
5. **性能优化**：包含碰撞检测、边优化等性能优化措施

建议后续优化方向：
1. 完善依赖注入机制
2. 增强配置管理
3. 实现更高级的缓存策略
4. 优化大型图谱的性能表现
