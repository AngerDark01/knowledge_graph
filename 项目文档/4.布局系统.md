# 布局系统

## 布局系统 (src/services/layout)

布局系统是知识图谱编辑器的重要组成部分，负责自动布局节点、优化视觉效果、防止节点重叠。

### 核心架构

#### LayoutManager.ts
布局管理器是整个布局系统的中心协调器。

**职责**:
- 管理和注册不同的布局策略
- 根据选项智能选择布局策略
- 提供统一的布局应用接口
- 处理布局过程中的异步操作
- 支持布局操作的取消

**主要功能**:
```typescript
interface ILayoutManager {
  applyLayout(nodes: (Node | Group)[], edges: Edge[], options?: LayoutOptions): Promise<LayoutResult>;
  registerStrategy(name: string, strategy: ILayoutStrategy): void;
  getSupportedStrategies(): string[];
  cancelCurrentOperation(): void;
  setConfig(config: Partial<typeof LAYOUT_CONFIG>): void;
  getConfig(): typeof LAYOUT_CONFIG;
}
```

**智能策略选择**:
LayoutManager 实现了智能策略选择机制，根据选项自动选择最适合的策略：
- 如果显式指定 `strategy`，直接使用该策略
- 如果 `layoutMode === 'recursive'`，使用递归布局策略
- 如果提供了 `targetGroupId`，使用群组布局策略
- 默认使用画布布局策略

### 类型定义 (src/services/layout/types)

#### layoutTypes.ts
定义布局系统所需的所有类型。

**核心类型**:
```typescript
export interface LayoutOptions {
  strategy?: string;
  animate?: boolean;
  targetGroupId?: string | null;  // 指定目标群组ID
  layoutScope?: 'canvas' | 'group';  // 布局范围类型
  layoutMode?: 'normal' | 'recursive';  // 布局模式
  onProgress?: (progress: { currentLevel: number; totalLevels: number; processedNodes: number; totalNodes: number }) => void;
  [key: string]: any;
}

export interface LayoutResult {
  success: boolean;
  nodes: Map<string, { x: number; y: number }>;
  edges: Map<string, any>;
  errors: string[];
  stats: {
    duration: number;
    iterations: number;
    collisions: number;
  };
}

export interface ILayoutStrategy {
  name: string;
  id: string;

  applyLayout(
    nodes: LayoutNode[],
    edges: LayoutEdge[],
    options?: LayoutOptions
  ): Promise<LayoutResult>;

  validateConfig(config: any): boolean;
}
```

### 工具函数 (src/services/layout/utils)

#### GeometryUtils.ts
提供几何计算相关的工具函数。

**主要功能**:
- `distance()`: 计算两点间的欧几里得距离
- `isOverlapping()`: 判断两个矩形是否重叠（AABB碰撞检测）
- `getCenter()`: 计算矩形中心点
- `getBounds()`: 获取节点边界
- `getEnclosingBounds()`: 计算包围所有节点的边界框
- `getMTV()`: 计算两个矩形的最小平移向量（用于碰撞解决）

#### CoordinateTransformer.ts
提供坐标系统之间的转换功能。

**主要功能**:
- `toAbsolute()`: 将相对坐标转换为绝对坐标
- `toQuadrantFour()`: 将节点转换到第四象限（确保所有坐标为正数）
- `calculateGroupAnchor()`: 计算群组的锚点（内容区域左上角）
- `relativeToAbsoluteForGroup()`: 批量转换：从相对坐标到绝对坐标（群组场景）

#### CollisionResolver.ts
使用迭代式排斥力算法解决节点重叠问题。

**主要功能**:
- `resolve()`: 解决节点之间的碰撞
- `countCollisions()`: 计算碰撞数量
- `hasCollisions()`: 检查是否存在碰撞

#### GroupSizeAdjuster.ts
根据子节点的实际占用空间自动调整父群组的尺寸。

**主要功能**:
- `adjustSingleGroup()`: 调整单个群组的大小
- `adjustMultipleGroups()`: 批量调整多个群组的大小
- `calculateChildrenRelativeBounds()`: 计算子节点相对于父群组的边界

#### NestingTreeBuilder.ts
分析节点关系，构建嵌套层级结构，计算嵌套深度。

**主要功能**:
- `calculateDepth()`: 递归计算节点的嵌套深度
- `detectCircular()`: 检测是否存在循环嵌套
- `groupNodesByDepth()`: 按深度对节点分组
- `buildTree()`: 构建嵌套层级树
- `getGroupsAtDepth()`: 获取指定深度的所有群组节点
- `getMaxDepth()`: 获取最大嵌套深度

### 布局策略 (src/services/layout/strategies)

布局策略实现了不同的自动布局算法，每种策略针对不同的使用场景。

#### CanvasLayoutStrategy.ts
画布布局策略，对所有顶层节点（没有groupId）进行布局。

**特点**:
- 使用居中布局算法（以原点为中心）
- 处理嵌套节点位置更新（子节点跟随父节点移动）
- 优化边的连接点
- 高权重节点优先排列在中心位置

**功能**:
- 筛选顶层节点
- 使用GridCenterAlgorithm计算居中布局
- 碰撞检测与解决
- 更新嵌套节点位置（保持相对位置）
- 优化边的连接点

#### GroupLayoutStrategy.ts
群组内布局策略，对指定群组的直接子节点进行布局。

**特点**:
- 使用固定锚点（群组左上角 + padding）
- 自动调整群组大小以适应子节点
- 递归更新嵌套子节点位置（保持相对位置）

**功能**:
- 筛选目标群组的直接子节点（去重）
- 使用GridAlgorithm计算相对布局（从左上角开始）
- 转换为绝对坐标（基于群组锚点）
- 碰撞检测与解决
- 递归更新嵌套子节点位置
- 自适应调整群组大小

#### RecursiveLayoutStrategy.ts
递归布局策略，对所有嵌套层级的节点进行递归布局。

**特点**:
- 从最深层群组开始，逐层向上布局
- 作为GroupLayoutStrategy和CanvasLayoutStrategy的编排器

**功能**:
- 构建嵌套层级树（使用NestingTreeBuilder）
- 从最深层到第1层，依次布局每个群组（调用GroupLayoutStrategy）
- 布局顶层节点（调用CanvasLayoutStrategy）
- 从深到浅调整所有群组大小
- 优化所有边

### 算法层 (src/services/layout/algorithms)

#### GridAlgorithm.ts
网格布局算法，从左上角开始布局，所有坐标为正数（第四象限），适用于群组内部布局。

#### GridCenterAlgorithm.ts
网格中心布局算法，以原点为中心布局，可能产生负坐标，适用于全画布布局（顶层节点）。

#### EdgeOptimizer.ts
边优化算法，优化边的连接点以提供更好的视觉效果。

### API 实现

布局系统通过 API 端点提供给前端调用：

- `/api/layout` - POST 请求应用布局算法
- `/api/layout?action=strategies` - GET 请求获取支持的策略列表

### UI 集成

#### LayoutControl.tsx
布局控制组件，提供用户界面来选择和应用不同的布局策略。

**功能**:
- 选择布局策略（应用布局、布局群组内部、递归布局全部）
- 应用布局算法
- 显示布局进度
- 提供布局配置选项

**集成**:
- 在 Toolbar 中作为独立控件
- 与 GraphStore 集成
- 调用 LayoutManager 执行布局
