# 布局系统

## 布局系统 (src/services/layout)

布局系统是知识图谱编辑器的重要组成部分，负责自动布局节点、优化视觉效果、防止节点重叠。

### 核心架构

#### LayoutManager.ts
布局管理器是整个布局系统的中心协调器。

**职责**:
- 管理和注册不同的布局策略
- 提供统一的布局应用接口
- 处理布局过程中的异步操作
- 支持布局操作的取消

**主要功能**:
```typescript
interface ILayoutManager {
  applyLayout(nodes: (Node | Group)[], edges: Edge[], options?: LayoutOptions): Promise<LayoutResult>;
  registerStrategy(name: string, strategy: ILayoutStrategy): void;
  getSupportedStrategies(): string[];
  cancelCurrentOperation(): void;
  setConfig(config: Partial<typeof LAYOUT_CONFIG>): void;
  getConfig(): typeof LAYOUT_CONFIG;
}
```

### 类型定义 (src/services/layout/types)

#### layoutTypes.ts
定义布局系统所需的所有类型。

**核心类型**:
```typescript
export interface LayoutOptions {
  strategy?: string;
  animate?: boolean;
  onProgress?: (progress: { currentLevel: number; totalLevels: number; processedNodes: number; totalNodes: number }) => void;
  [key: string]: any;
}

export interface LayoutResult {
  success: boolean;
  nodes: Map<string, { x: number; y: number }>;
  edges: Map<string, any>;
  errors: string[];
  stats: {
    duration: number;
    iterations: number;
    collisions: number;
  };
}

export interface ILayoutStrategy {
  name: string;
  id: string;

  applyLayout(
    nodes: LayoutNode[],
    edges: LayoutEdge[],
    options?: LayoutOptions
  ): Promise<LayoutResult>;

  validateConfig(config: any): boolean;
}
```

### 工具函数 (src/services/layout/utils)

#### GeometryUtils.ts
提供几何计算相关的工具函数。

**主要功能**:
- `distance()`: 计算两点间的欧几里得距离
- `isOverlapping()`: 判断两个矩形是否重叠（AABB碰撞检测）
- `getCenter()`: 计算矩形中心点
- `getBounds()`: 获取节点边界
- `getEnclosingBounds()`: 计算包围所有节点的边界框
- `getMTV()`: 计算两个矩形的最小平移向量（用于碰撞解决）

### 布局策略 (src/services/layout/strategies)

布局策略实现了不同的自动布局算法，每种策略针对不同的使用场景。

#### GridCenterLayoutStrategy.ts
网格中心布局策略，将高权重节点放置在中心，其他节点围绕排列。

**特点**:
- 基于权重的节点排列
- 避免节点重叠
- 保持嵌套节点的相对位置
- 支持多层嵌套结构（Group嵌套Group，再嵌套Node）
- 父节点移动时保持所有嵌套子节点递归的稳定性

**功能**:
- 标准化节点尺寸
- 计算节点权重
- 按网格排列节点
- 解决节点碰撞
- 递归计算嵌套节点位置
- 多层嵌套边界约束

**核心方法**:
- `updateNestedNodePositions()`: 根据父节点的新位置更新所有嵌套节点的相对位置，支持多层嵌套结构
- `calculateAbsolutePosition()`: 递归计算嵌套节点的绝对位置，查找节点的所有祖先群组，计算其相对于最顶层父群组的最终位置
- `getOriginalAbsolutePosition()`: 递归计算节点的原始绝对位置，获取节点在原始状态下的绝对位置（考虑到所有祖先的位置）
- `adjustNestedNodesWithinBounds()`: 调整嵌套节点确保它们在父群组边界内，同时保持彼此间的相对位置，支持多层嵌套

#### GroupAwareLayoutStrategy.ts
组感知布局策略，优先处理组内节点布局，保持组结构清晰。

**特点**:
- 尊重组结构
- 组内网格布局
- 组间间距控制

**功能**:
- 按父组对节点分组
- 组内节点布局
- 组间碰撞解决

#### EdgeOptimizationStrategy.ts
边优化策略，优化边的连接点和路径，减少交叉。

**特点**:
- 智能连接点选择
- 边路径优化
- 支持多种边样式

**功能**:
- 计算最佳连接点
- 优化边路径
- 支持直线、正交、曲线路径

#### CompositeLayoutStrategy.ts
综合布局策略，结合节点布局、组布局和边优化的综合方案。

**特点**:
- 多策略组合
- 统一处理节点和边
- 可配置的策略组合

### API 实现

布局系统通过 API 端点提供给前端调用：

- `/api/layout` - POST 请求应用布局算法
- `/api/layout?action=strategies` - GET 请求获取支持的策略列表

### UI 集成

#### LayoutControl.tsx
布局控制组件，提供用户界面来选择和应用不同的布局策略。

**功能**:
- 选择布局策略
- 应用布局算法
- 显示布局进度
- 提供布局配置选项

**集成**:
- 在 Toolbar 中作为独立控件
- 与 GraphStore 集成
- 调用 LayoutManager 执行布局
