# 布局系统开发指南

## 概述

布局系统是知识图谱编辑器的重要组成部分，负责自动排列节点、优化视觉效果、防止节点重叠。系统支持多层嵌套结构（Group嵌套Group，再嵌套Node），确保父节点移动时所有嵌套子节点保持相对位置不变。

## 核心组件

### 1. LayoutManager (src/services/layout/LayoutManager.ts)

- 作为布局系统的中心协调器
- 管理和注册不同的布局策略
- 提供统一的布局应用接口
- 处理布局过程中的异步操作

### 2. 布局策略 (src/services/layout/strategies/)

实现不同的自动布局算法，包括：

- **GridCenterLayoutStrategy**: 网格中心布局，优先处理高权重节点
- **GroupAwareLayoutStrategy**: 组感知布局，优先处理组内节点
- **EdgeOptimizationStrategy**: 边优化策略，优化连接点
- **CompositeLayoutStrategy**: 综合策略，结合多个策略

## 多层嵌套支持的实现

### 核心概念

在多层嵌套结构中，系统需要确保：
1. 父节点移动时，所有嵌套层级的子节点相应移动
2. 每个节点相对于其直接父群组的相对位置保持不变
3. 所有嵌套节点在边界约束下仍保持适当位置

### 关键方法（位于 GridCenterLayoutStrategy.ts）

#### 1. `updateNestedNodePositions(originalNodes, positionedNodes)`

这个方法是多层嵌套支持的核心，它：
- 分离顶层节点和嵌套节点
- 为每个嵌套节点计算其绝对位置
- 创建节点映射以提高查找性能

#### 2. `calculateAbsolutePosition(node, originalNodeMap, positionedNodeMap)`

这是多层嵌套支持的关键递归方法：
- 递归计算嵌套节点的绝对位置
- 查找节点的所有祖先群组
- 计算节点相对于最顶层父群组的最终位置

实现逻辑：
```typescript
private calculateAbsolutePosition(
  node: Node | Group,
  originalNodeMap: Map<string, Node | Group>,
  positionedNodeMap: Map<string, Node | Group>
): { x: number; y: number } {
  // 1. 如果节点没有父群组，返回其在positionedNodes中的位置
  if (!('groupId' in node) || !node.groupId) {
    const positionedNode = positionedNodeMap.get(node.id);
    return positionedNode ? positionedNode.position : node.position;
  }

  // 2. 获取原始父群组
  const originalParentGroup = originalNodeMap.get(node.groupId) as Group;
  if (!originalParentGroup) {
    return node.position;
  }

  // 3. 递归计算父群组的新绝对位置
  const parentAbsolutePosition = this.calculateAbsolutePosition(
    originalParentGroup,
    originalNodeMap,
    positionedNodeMap
  );

  // 4. 获取原始父群组的位置
  const originalParentAbsolutePosition = this.getOriginalAbsolutePosition(
    originalParentGroup,
    originalNodeMap
  );

  // 5. 计算节点相对于原始父群组的相对位置
  const relativeX = node.position.x - originalParentGroup.position.x;
  const relativeY = node.position.y - originalParentGroup.position.y;

  // 6. 基于父群组的新绝对位置和节点的相对位置，计算节点的新绝对位置
  const newAbsoluteX = parentAbsolutePosition.x + relativeX;
  const newAbsoluteY = parentAbsolutePosition.y + relativeY;

  return { x: newAbsoluteX, y: newAbsoluteY };
}
```

#### 3. `getOriginalAbsolutePosition(node, originalNodeMap)`

这个辅助方法：
- 递归计算节点的原始绝对位置
- 获取节点在原始状态下的绝对位置（考虑到所有祖先的位置）

## 性能优化

### 1. 查找性能优化
- 使用 Map 数据结构创建节点映射，提高查找性能
- 避免在循环中进行线性查找

### 2. 边界约束优化
- `adjustNestedNodesWithinBounds` 方法使用迭代方法处理边界约束
- 设置最大迭代次数（默认10次）防止无限循环

### 3. 递归算法优化
- 对递归深度进行限制
- 避免不必要的重复计算

## 常见问题与调试

### 1. 检查多层嵌套是否正常工作

验证以下几点：
- 父节点移动时，所有嵌套层级的子节点是否相应移动
- 每个子节点相对于其直接父群组的相对位置是否保持不变
- 所有节点是否仍在其父群组边界内

### 2. 调试方法

在需要调试时，可以：

1. 在关键方法中添加日志：
```typescript
console.log('Processing node:', node.id, 'with groupId:', node.groupId);
```

2. 验证递归计算是否正确：
```typescript
console.log('Original relative position:', relativeX, relativeY);
console.log('New absolute position:', newAbsoluteX, newAbsoluteY);
```

## 扩展和维护

### 添加新的布局策略

1. 实现 ILayoutStrategy 接口
2. 实现 applyLayout 方法
3. 在 LayoutManager 中注册新策略

### 修改多层嵌套行为

如需修改嵌套节点的处理方式，重点关注：
- `calculateAbsolutePosition` 方法的递归逻辑
- `getOriginalAbsolutePosition` 中的原始位置计算
- `adjustNestedNodesWithinBounds` 中的边界处理

## 总结

布局系统的多层嵌套支持通过递归计算节点的绝对位置，确保了父节点移动时所有嵌套子节点能够保持相对位置。该实现既保证了功能的完整性，又考虑了性能优化，通过节点映射、迭代边界约束和递归深度控制等手段，提供了高效的多层嵌套支持。
