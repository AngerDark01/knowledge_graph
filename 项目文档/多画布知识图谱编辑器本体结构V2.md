# 多画布知识图谱编辑器项目本体结构V2

## 1. 项目概述
这是一个基于 Next.js 和 React Flow 的多画布知识图谱编辑器，支持嵌套群组、节点转换、自动布局和多画布管理等功能。采用模块化设计，包含项目级本地存储、工作空间管理、三栏UI布局等特性，通过Git同步数据。

## 3. 主要数据流向
1. **应用启动**: `src/app/page.tsx` → `StorageManager.loadWorkspace()` (通过API路由) → 初始化 `workspaceStore` → 加载当前画布数据到 `graphStore` → 渲染 `WorkspaceLayout`
2. **画布切换**: 用户点击画布 → `CanvasTreeItem.onClick()` → `workspaceStore.switchCanvas()` → 保存当前画布状态 → 加载目标画布数据 → `graphStore.replaceGraphData()` → `GraphPageContent` 重新渲染
3. **图数据变更**: 图数据变更 → `graphStore` 更新 → `persistenceMiddleware` 监听变更 → 防抖处理 → `StorageManager.saveCanvas()` (通过API路由) → 写入项目文件
4. **节点生命周期**: 创建 → 添加到 store → 渲染到 ReactFlow → 交互 → 更新状态
5. **布局处理**: API 调用 → LayoutManager → 布局策略 → ELK 转换 → 位置更新

## 4. 架构模式
- **状态管理**: 使用 Zustand 进行全局状态管理，采用切片模式，分为 `workspaceStore` 和 `graphStore`
- **UI 框架**: React + React Flow 实现图可视化，三栏布局实现工作空间管理
- **后端服务**: Next.js App Router + API Routes 实现安全的文件系统访问
- **类型安全**: 使用 Zod 进行数据验证，TypeScript 保证类型安全
- **存储抽象**: 采用适配器模式，便于未来切换到其他存储方式
- **布局算法**: ELK.js 用于自动布局
- **样式**: Tailwind CSS + 自定义样式

## 5. 关键依赖关系
- 项目核心依赖于 React Flow 进行图可视化
- Zustand 用于状态管理，是项目数据流的核心
- ELK.js 用于自动布局，是项目图布局的关键
- Next.js 作为主要框架，处理路由和 API
- Zod 确保数据验证和类型安全
- 通过 `StorageManager` 和 API Routes 服务实现安全的本地文件访问
- 采用三栏布局支持多画布管理和工作空间功能
