## 1. 概述

本文档详细描述了知识图谱编辑器的永久化存储表结构，主要用于支持多画布管理功能。系统采用JSON文件方式将数据存储在项目文件中，便于通过Git进行同步。

### 1.1 设计原则

- **模块化设计**: 数据模型结构清晰，各模块职责明确
- **类型安全**: 使用Zod进行数据验证，确保数据一致性
- **可扩展性**: 预留字段和接口，便于未来功能扩展
- **性能考虑**: 优化数据访问模式，减少不必要的数据加载

### 1.2 存储策略

- 采用单文件存储方式，将完整工作区数据存储在 `kg-editor:workspace.json` 文件中
- 使用版本控制机制，便于数据迁移和兼容性管理
- 支持增量更新，仅在数据变化时保存

## 5. 数据关系

### 5.1 画布与节点关系

1. **Canvas → Nodes/Edges**: 一个画布包含多个节点和边 (`1:N`)
2. **Node/Edge → Canvas**: 一个节点/边属于一个画布 (`N:1`)

### 5.2 画布层级关系

1. **Canvas → Parent Canvas**: 一个画布最多有一个父画布 (`N:1`)
2. **Canvas → Child Canvases**: 一个画布可以有多个子画布 (`1:N`)

### 5.3 群组与节点关系

1. **Group → Child Nodes**: 一个群组可以包含多个节点 (`1:N`)
2. **Node → Group**: 一个节点最多属于一个群组 (`N:1`)

### 5.4 节点与边关系

1. **Edge → Source Node**: 一条边有一个源节点 (`N:1`)
2. **Edge → Target Node**: 一条边有一个目标节点 (`N:1`)
3. **Node → Edges**: 一个节点可以有多个关联边 (`1:N`)

### 5.5 数据流向

```
Canvas
  ↓ (包含)
(graphData.nodes, graphData.edges)
  ↓
Nodes/Edges
  ↓ (通过groupId关联)
Groups
  ↓ (嵌套结构)
Nested Nodes/Groups
```
