# 知识图谱编辑器后端服务架构方案

## 1. 架构概述

参考 Dify 的"蜂巢架构"（Beehive Architecture），将项目重构为**前后端分离的模块化架构**，每个模块独立且可扩展。

### 1.1 核心设计原则

- **前后端分离**：前端专注于UI渲染和用户交互，后端处理业务逻辑和数据持久化
- **模块化设计**：参考 Dify 的蜂巢架构，每个模块独立且可替换
- **API优先**：所有功能通过 RESTful API 暴露
- **水平扩展**：支持微服务化拆分和容器化部署
- **插件化扩展**：预留插件系统接口，支持第三方扩展

## 2. 技术栈选择

### 2.1 后端技术栈

**方案A：Node.js/TypeScript 栈（推荐）**
- **框架**：Nest.js（企业级 Node.js 框架，类似 Spring Boot）
- **语言**：TypeScript（与前端共享类型定义）
- **数据库**：PostgreSQL + TypeORM
- **缓存**：Redis
- **消息队列**：BullMQ（基于 Redis）
- **文件存储**：MinIO / S3
- **搜索引擎**：Elasticsearch（可选，用于全文搜索）

**优势**：
- 与前端共享 TypeScript 类型系统，减少重复定义
- 团队技术栈统一，降低学习成本
- 性能优秀，生态丰富
- Nest.js 提供完整的依赖注入、模块化、测试支持

**方案B：Python/Flask 栈**
- **框架**：Flask + Flask-RESTful
- **语言**：Python 3.11+
- **ORM**：SQLAlchemy
- **数据库**：PostgreSQL
- **缓存**：Redis
- **任务队列**：Celery

**优势**：
- 与 AI/ML 生态集成更容易（如果未来需要知识图谱分析、图算法等）
- 参考 Dify 的实现，社区方案成熟

### 2.2 前端技术栈（保持不变）

- **框架**：Next.js 16（作为纯前端应用，关闭 SSR）
- **UI库**：React 19 + React Flow
- **状态管理**：Zustand
- **样式**：Tailwind CSS

### 2.3 基础设施

- **容器化**：Docker + Docker Compose
- **反向代理**：Nginx
- **监控**：Prometheus + Grafana（可选）
- **日志**：ELK Stack（可选）

## 4. 数据库设计

### 4.1 核心实体关系

```
User (用户)
  ├─ 1:N → Workspace (工作空间)
  └─ N:M → Workspace (通过 WorkspaceMember)

Workspace (工作空间)
  ├─ 1:N → Canvas (画布)
  ├─ N:M → User (通过 WorkspaceMember)
  └─ 1:N → WorkspaceMember (成员关系)

Canvas (画布)
  ├─ 1:1 → Workspace
  ├─ 1:N → Node (节点)
  ├─ 1:N → Edge (边)
  └─ 1:N → CanvasSnapshot (快照/版本)

Node (节点)
  ├─ 1:1 → Canvas
  └─ 1:1 → Node (自引用，parentId)

Edge (边)
  ├─ 1:1 → Canvas
  ├─ 1:1 → Node (source)
  └─ 1:1 → Node (target)

CanvasSnapshot (画布快照)
  └─ 1:1 → Canvas
```

### 4.2 数据库表设计

#### 用户表 (users)
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  username VARCHAR(255) UNIQUE NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  avatar_url VARCHAR(500),
  display_name VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  last_login_at TIMESTAMP
);
```

#### 工作空间表 (workspaces)
```sql
CREATE TABLE workspaces (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(255) NOT NULL,
  description TEXT,
  icon VARCHAR(50),
  owner_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  is_personal BOOLEAN DEFAULT false,
  settings JSONB DEFAULT '{}',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_workspaces_owner_id ON workspaces(owner_id);
```

#### 工作空间成员表 (workspace_members)
```sql
CREATE TABLE workspace_members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  workspace_id UUID NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  role VARCHAR(50) NOT NULL DEFAULT 'viewer', -- owner, editor, viewer
  joined_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(workspace_id, user_id)
);

CREATE INDEX idx_workspace_members_workspace_id ON workspace_members(workspace_id);
CREATE INDEX idx_workspace_members_user_id ON workspace_members(user_id);
```

#### 画布表 (canvases)
```sql
CREATE TABLE canvases (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  workspace_id UUID NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
  parent_id UUID REFERENCES canvases(id) ON DELETE CASCADE,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  icon VARCHAR(50),
  position INTEGER DEFAULT 0, -- 排序位置
  view_state JSONB, -- 视口状态（缩放、平移等）
  settings JSONB DEFAULT '{}',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_canvases_workspace_id ON canvases(workspace_id);
CREATE INDEX idx_canvases_parent_id ON canvases(parent_id);
```

#### 节点表 (nodes)
```sql
CREATE TABLE nodes (
  id VARCHAR(255) PRIMARY KEY, -- 使用 nanoid
  canvas_id UUID NOT NULL REFERENCES canvases(id) ON DELETE CASCADE,
  parent_id VARCHAR(255) REFERENCES nodes(id) ON DELETE SET NULL,
  type VARCHAR(50) NOT NULL, -- 'note', 'group'
  position_x FLOAT NOT NULL,
  position_y FLOAT NOT NULL,
  width FLOAT,
  height FLOAT,
  z_index INTEGER DEFAULT 0,
  data JSONB NOT NULL, -- 节点数据（label, content, tags等）
  style JSONB, -- 样式数据
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_nodes_canvas_id ON nodes(canvas_id);
CREATE INDEX idx_nodes_parent_id ON nodes(parent_id);
CREATE INDEX idx_nodes_type ON nodes(type);
```

#### 边表 (edges)
```sql
CREATE TABLE edges (
  id VARCHAR(255) PRIMARY KEY, -- 使用 nanoid
  canvas_id UUID NOT NULL REFERENCES canvases(id) ON DELETE CASCADE,
  source_id VARCHAR(255) NOT NULL REFERENCES nodes(id) ON DELETE CASCADE,
  target_id VARCHAR(255) NOT NULL REFERENCES nodes(id) ON DELETE CASCADE,
  type VARCHAR(50) DEFAULT 'default',
  label VARCHAR(255),
  style JSONB,
  data JSONB,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_edges_canvas_id ON edges(canvas_id);
CREATE INDEX idx_edges_source_id ON edges(source_id);
CREATE INDEX idx_edges_target_id ON edges(target_id);
```

#### 画布快照表 (canvas_snapshots)
```sql
CREATE TABLE canvas_snapshots (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  canvas_id UUID NOT NULL REFERENCES canvases(id) ON DELETE CASCADE,
  version INTEGER NOT NULL,
  snapshot_data JSONB NOT NULL, -- 完整的画布数据
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW(),
  description TEXT
);

CREATE INDEX idx_canvas_snapshots_canvas_id ON canvas_snapshots(canvas_id);
CREATE INDEX idx_canvas_snapshots_created_at ON canvas_snapshots(created_at);
```

## 5. API 设计

### 5.1 API 规范

- **基础URL**: `http://api.domain.com/v1`
- **认证方式**: JWT (Bearer Token)
- **响应格式**: JSON
- **状态码**: 标准 HTTP 状态码

#### 统一响应格式
```typescript
// 成功响应
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}

// 错误响应
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述",
    "details": { ... }
  }
}
```

### 5.2 API 端点设计

#### 认证模块
```
POST   /auth/register          # 注册
POST   /auth/login             # 登录
POST   /auth/logout            # 登出
POST   /auth/refresh           # 刷新令牌
GET    /auth/me                # 获取当前用户信息
```

#### 用户模块
```
GET    /users/:id              # 获取用户信息
```

#### 工作空间模块

### 6.3 Docker Compose 配置示例

```yaml
version: '3.8'

services:
  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./docker/nginx/nginx.conf:/etc/nginx/nginx.conf
      - ./frontend/out:/usr/share/nginx/html
    depends_on:
      - backend
      - frontend

  # 前端服务
  frontend:
    build:
      context: ./frontend
      dockerfile: ../docker/frontend/Dockerfile
    environment:
      - NEXT_PUBLIC_API_URL=http://nginx/api

  # 后端服务
  backend:
    build:
      context: ./backend
      dockerfile: ../docker/backend/Dockerfile
    ports:
      - "4000:4000"
    environment:
      - DATABASE_URL=postgresql://user:password@postgres:5432/knowledge_graph
      - REDIS_URL=redis://redis:6379
      - MINIO_ENDPOINT=minio:9000
    depends_on:
      - postgres
      - redis
      - minio

  # PostgreSQL 数据库
  postgres:
    image: postgres:16-alpine
    ports:
      - "5432:5432"
    environment:
      - POSTGRES_DB=knowledge_graph
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data

  # Redis 缓存
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
```

## 8. 优势与收益

### 8.1 技术优势
- **可扩展性**：模块化设计，易于扩展新功能
- **性能**：数据库索引、Redis 缓存、连接池优化
- **可靠性**：数据库事务、错误处理、日志记录
- **安全性**：JWT 认证、权限控制、SQL 注入防护

### 8.2 业务优势
- **多用户支持**：支持团队协作
- **数据安全**：数据库持久化，自动备份
- **版本控制**：画布快照，可回溯历史
- **实时协作**：WebSocket 实时同步

### 8.3 运维优势
- **容器化部署**：Docker + Docker Compose
- **水平扩展**：可部署多个后端实例
- **监控告警**：Prometheus + Grafana
- **日志管理**：ELK Stack

## 9. 技术选型对比

| 维度 | Node.js/Nest.js | Python/Flask |
|-----|----------------|--------------|
| 类型系统 | ✅ TypeScript，与前端共享 | ❌ 需要重复定义 |
| 学习成本 | ✅ 团队统一技术栈 | ❌ 需要学习 Python |
| 性能 | ✅ 异步 I/O，高并发 | ⚠️ GIL 限制 |
| 生态 | ✅ npm 生态丰富 | ✅ PyPI 生态成熟 |
| AI/ML 集成 | ⚠️ 需要额外工作 | ✅ 原生支持 |
| 企业级框架 | ✅ Nest.js 成熟 | ⚠️ Flask 相对轻量 |
| 依赖注入 | ✅ 内置支持 | ❌ 需要额外库 |
| 测试支持 | ✅ Jest 成熟 | ✅ pytest 优秀 |

**推荐：Node.js/Nest.js**（理由：类型系统统一、团队技术栈一致、企业级支持完善）
