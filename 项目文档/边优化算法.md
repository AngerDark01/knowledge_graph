# 边优化算法

## 概述

边优化算法是知识图谱编辑器中的一个重要功能，主要用于在节点位置发生变化后，智能地调整边的连接点（handles），以提供更好的视觉效果和用户体验。该算法通过计算节点之间的相对位置关系，确定最佳的连接点方向（上、右、下、左），从而减少边的交叉和重叠。

## 核心设计原则

1. **基于节点位置**：边的连接点选择完全基于节点的当前位置和尺寸，与边的出入顺序无关
2. **视觉优化**：通过选择合理的连接点，使边的路径更加清晰，减少视觉混乱
3. **性能优化**：支持批量优化和增量优化，避免不必要的重复计算
4. **实时响应**：支持节点拖拽后的实时优化

## 算法逻辑

### 1. 角度计算

边优化算法首先计算源节点和目标节点的中心点，然后计算它们之间的角度：

```typescript
// 计算两个节点中心点
const sourceCenter = GeometryUtils.getCenter({
  x: sourceNode.position.x,
  y: sourceNode.position.y,
  width: sourceNode.width || LAYOUT_CONFIG.nodeSize.defaultNode.width,
  height: sourceNode.height || LAYOUT_CONFIG.nodeSize.defaultNode.height
});

const targetCenter = GeometryUtils.getCenter({
  x: targetNode.position.x,
  y: targetNode.position.y,
  width: targetNode.width || LAYOUT_CONFIG.nodeSize.defaultNode.width,
  height: targetNode.height || LAYOUT_CONFIG.nodeSize.defaultNode.height
});

// 计算源节点到目标节点的角度
const angle = Math.atan2(
  targetCenter.y - sourceCenter.y,
  targetCenter.x - sourceCenter.x
);
```

### 2. 连接点选择策略

根据计算出的角度，算法将角度范围划分为四个象限，每个象限对应一个连接点方向：

- **右象限**：-π/4 ≤ angle < π/4 → 源节点使用"右"连接点，目标节点使用"左"连接点
- **下象限**：π/4 ≤ angle < 3π/4 → 源节点使用"下"连接点，目标节点使用"上"连接点
- **上象限**：-3π/4 ≤ angle < -π/4 → 源节点使用"上"连接点，目标节点使用"下"连接点
- **左象限**：其他情况 → 源节点使用"左"连接点，目标节点使用"右"连接点

### 3. 连接点选择逻辑

```typescript
// 选择源节点的连接点
if (normalizedAngle >= -QUADRANT && normalizedAngle < QUADRANT) {
  sourceHandle = 'right';  // 目标在右侧
} else if (normalizedAngle >= QUADRANT && normalizedAngle < 3 * QUADRANT) {
  sourceHandle = 'bottom'; // 目标在下方
} else if (normalizedAngle >= -3 * QUADRANT && normalizedAngle < -QUADRANT) {
  sourceHandle = 'top';    // 目标在上方
} else {
  sourceHandle = 'left';   // 目标在左侧
}

// 优化目标节点的连接点选择逻辑
// 根据源节点和目标节点的相对位置来选择目标节点的连接点
if (normalizedAngle >= -QUADRANT && normalizedAngle < QUADRANT) {
  // 目标在源节点的右侧
  targetHandle = 'left';  // 所以连接到目标节点的左侧
} else if (normalizedAngle >= QUADRANT && normalizedAngle < 3 * QUADRANT) {
  // 目标在源节点的下方
  targetHandle = 'top';   // 所以连接到目标节点的顶部
} else if (normalizedAngle >= -3 * QUADRANT && normalizedAngle < -QUADRANT) {
  // 目标在源节点的上方
  targetHandle = 'bottom'; // 所以连接到目标节点的底部
} else {
  // 目标在源节点的左侧
  targetHandle = 'right';  // 所以连接到目标节点的右侧
}
```

## 配置参数

边优化算法的行为由 `EDGE_OPTIMIZATION_CONFIG` 配置对象控制：

```typescript
export const EDGE_OPTIMIZATION_CONFIG = {
  /** 是否启用边优化 */
  ENABLED: true,
  /** 角度阈值（用于确定连接点方向） */
  ANGLE_THRESHOLDS: {
    /** π/4，用于判断是水平还是垂直方向 */
    QUADRANT: Math.PI / 4,
  },
  /** 防抖延迟（毫秒） - 拖拽时避免频繁计算 */
  DEBOUNCE_DELAY: 100,
  /** 批量优化时的性能阈值 */
  PERFORMANCE: {
    /** 边数量超过此值时启用性能优化模式 */
    BATCH_THRESHOLD: 100,
    /** 使用 requestIdleCallback 的边数量阈值 */
    IDLE_CALLBACK_THRESHOLD: 50,
  },
} as const;
```

## 实现方法

### 1. 单条边优化

`calculateBestHandles(sourceNode, targetNode)` 方法为单条边计算最佳连接点。

### 2. 批量优化

`optimizeEdgeHandles(nodes, edges)` 方法批量优化所有边的连接点。

### 3. 增量优化

`optimizeBatch(nodes, edges, affectedNodeIds?)` 方法支持增量优化，可以只优化受特定节点影响的边。

## 应用时机

### 1. 布局后优化

在应用布局算法后，系统会自动对所有边进行一次优化：

```typescript
// 额外的边优化：确保所有边的连接点基于最终节点位置进行了优化
const finalNodes = useGraphStore.getState().getNodes();
const finalEdges = useGraphStore.getState().getEdges();

import('@/services/layout/algorithms/EdgeOptimizer').then(({ EdgeOptimizer }) => {
  const edgeOptimizer = new EdgeOptimizer();
  const optimizedEdges = edgeOptimizer.optimizeEdgeHandles(finalNodes, finalEdges);
  
  // 更新所有边的连接点
  optimizedEdges.forEach((edgeData) => {
    updateEdge(edgeData.id, {
      sourceHandle: edgeData.sourceHandle,
      targetHandle: edgeData.targetHandle
    });
  });
});
```

### 2. 节点拖拽后优化

在节点拖拽结束后，系统会优化与拖拽节点相关的边：

```typescript
// 边优化防抖函数：在拖拽结束后优化相关边的连接点
const optimizeEdgesAfterDrag = useCallback((draggedNodeId: string) => {
  if (!EDGE_OPTIMIZATION_CONFIG.ENABLED) return;

  // 使用防抖机制避免频繁计算
  edgeOptimizeTimerRef.current = setTimeout(() => {
    const { nodes, edges: allEdges, updateEdge } = useGraphStore.getState();
    
    // 找到与拖拽节点相关的边
    const affectedNodeIds = new Set([draggedNodeId]);
    const affectedEdges = allEdges.filter(
      (edge) => edge.source === draggedNodeId || edge.target === draggedNodeId
    );

    if (affectedEdges.length === 0) return;

    // 使用批量优化
    const optimizedEdges = edgeOptimizerRef.current.optimizeBatch(
      nodes,
      allEdges,
      affectedNodeIds
    );

    // 只更新受影响的边
    affectedEdges.forEach((edge) => {
      const optimizedEdge = optimizedEdges.find((e) => e.id === edge.id);
      if (optimizedEdge && optimizedEdge.sourceHandle && optimizedEdge.targetHandle) {
        updateEdge(edge.id, {
          sourceHandle: optimizedEdge.sourceHandle,
          targetHandle: optimizedEdge.targetHandle,
        });
      }
    });
  }, EDGE_OPTIMIZATION_CONFIG.DEBOUNCE_DELAY);
}, []);
```

## 性能考虑

1. **防抖机制**：在节点拖拽等频繁操作中使用防抖，避免过度计算
2. **批量处理**：支持批量优化，减少重复操作
3. **阈值控制**：当边数量超过阈值时启用性能优化模式
4. **增量更新**：支持只优化受影响的边，而非全部边

## 未来改进方向

1. 考虑边的权重和重要性，优先优化重要边的连接点
2. 支持更复杂的边路径优化算法，如正交布局、曲线布局等
3. 增加对边交叉的检测和避免机制
4. 实现更智能的连接点分配，考虑节点形状和连接点可用性
