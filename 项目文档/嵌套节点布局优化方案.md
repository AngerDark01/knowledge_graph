# 知识图谱编辑器 - 嵌套节点布局优化方案

## 1. 现有实现分析

### 1.1 当前布局系统架构
- **GridCenterLayoutStrategy**: 当前仅对顶层节点进行网格布局
- **LayoutManager**: 负责协调布局策略的执行
- **LayoutControl**: UI组件，触发布局操作
- **Node/Group模型**: 支持嵌套关系（通过`groupId`字段）

### 1.2 核心数据结构
- **Node**: 普通节点，可包含`groupId`指向父群组
- **Group**: 群组节点，包含`nodeIds`数组记录子节点
- **边界约束**: 群组内节点不能超出群组边界

### 1.3 现有布局逻辑
- 只对没有`groupId`的顶层节点进行网格布局
- 嵌套节点位置通过`updateNestedNodePositions`和`calculateAbsolutePosition`保持相对位置
- 父群组移动时，所有子节点按相对位置同步移动

## 2. 现阶段问题识别

### 2.1 限制分析
- 当前仅支持对整个画布的顶层节点进行布局
- 无法对特定群组内部的节点进行独立布局
- 嵌套层次的布局选择受限

### 2.2 约束条件
- 子节点必须在父群组边界内
- 布局时要考虑群组尺寸限制
- 避免节点间过度拉伸导致视觉不佳

## 3. 第二阶段实现方案

### 3.1 核心设计思路

#### 3.1.1 层级布局策略
- 引入**目标群组指定**的布局策略
- 将布局操作从"全画布"模式扩展到"群组内部"模式
- 保持嵌套结构的完整性

#### 3.1.2 分层处理机制
- 选定群组时，只对该群组内的直接子节点进行布局
- 不影响该群组外的节点
- 不影响该群组内嵌套的更深层群组

### 3.2 技术架构设计

#### 3.2.1 新增布局策略

```typescript
// GridGroupLayoutStrategy - 专门用于群组内部布局的策略
class GridGroupLayoutStrategy implements ILayoutStrategy {
  applyLayout(
    nodes: (Node | Group)[],  // 所有节点
    edges: Edge[],            // 所有边
    options?: GridGroupLayoutOptions & { targetGroupId?: string }  // 指定目标群组ID
  ): Promise<LayoutResult>
}
```

#### 3.2.2 布局选项扩展
```typescript
interface GridGroupLayoutOptions extends LayoutOptions {
  targetGroupId?: string;      // 目标群组ID，如未指定则为顶层布局
  boundaryMode?: 'constrain' | 'expand' | 'auto'; // 边界处理模式
  spacing?: number;            // 群组内部节点间距
  padding?: { x: number; y: number }; // 群组内边距
}
```

### 3.3 核心算法实现

#### 3.3.1 目标节点筛选
```typescript
function getTargetNodesForLayout(
  allNodes: (Node | Group)[], 
  targetGroupId: string | null
): (Node | Group)[] {
  if (!targetGroupId) {
    // 顶层布局：返回所有顶层节点
    return allNodes.filter(node => !('groupId' in node) || !(node as Node).groupId);
  }
  
  // 群组内部布局：返回指定群组的直接子节点
  return allNodes.filter(node => 
    'groupId' in node && 
    (node as Node).groupId === targetGroupId
  );
}
```

#### 3.3.2 群组边界约束算法
```typescript
function applyBoundaryConstraints(
  positionedNodes: (Node | Group)[],
  parentGroup: Group,
  options: GridGroupLayoutOptions
): (Node | Group)[] {
  const groupBoundary = {
    left: parentGroup.position.x + (options.padding?.x || 20),
    top: parentGroup.position.y + (options.padding?.y || 20),
    right: parentGroup.position.x + (parentGroup.width || 300) - (options.padding?.x || 20),
    bottom: parentGroup.position.y + (parentGroup.height || 200) - (options.padding?.y || 20)
  };
  
  return positionedNodes.map(node => {
    // 确保节点在群组边界内
    const constrainedX = Math.max(
      groupBoundary.left + (node.width || 100) / 2,
      Math.min(groupBoundary.right - (node.width || 100) / 2, node.position.x)
    );
    
    const constrainedY = Math.max(
      groupBoundary.top + (node.height || 100) / 2,
      Math.min(groupBoundary.bottom - (node.height || 100) / 2, node.position.y)
    );
    
    return {
      ...node,
      position: { x: constrainedX, y: constrainedY }
    };
  });
}
```

#### 3.3.3 自适应间距计算
```typescript
function calculateOptimalSpacing(
  containerWidth: number,
  containerHeight: number,
  nodeCount: number,
  nodeWidth: number,
  nodeHeight: number
): { horizontalSpacing: number; verticalSpacing: number } {
  // 根据容器大小和节点数量计算最合适的间距
  // 防止节点间距离过大或过小
  const estimatedRows = Math.ceil(Math.sqrt(nodeCount));
  const estimatedCols = Math.ceil(nodeCount / estimatedRows);
  
  const availableWidth = containerWidth - (nodeWidth * estimatedCols);
  const availableHeight = containerHeight - (nodeHeight * estimatedRows);
  
  return {
    horizontalSpacing: Math.max(20, availableWidth / Math.max(1, estimatedCols - 1)),
    verticalSpacing: Math.max(20, availableHeight / Math.max(1, estimatedRows - 1))
  };
}
```

## 4. 约束处理策略

### 4.1 群组边界约束
- **自动边距**: 在群组边界内部保留固定边距
- **节点尺寸适配**: 根据节点大小调整布局密度
- **溢出检测**: 检测布局后节点是否超出群组边界

### 4.2 布局密度控制
- **最小间距**: 确保节点间有最小间距避免重叠
- **最大间距**: 限制节点间最大距离避免过度分散
- **自适应布局**: 根据群组尺寸和节点数量自动选择最优布局

### 4.3 嵌套结构完整性
- **层级分离**: 只处理指定层级的节点，不影响更深层嵌套
- **相对位置**: 保持节点相对于父群组的相对位置概念
- **边界自动调整**: 布局后自动调整群组边界以适应节点

## 5. 预期效果

### 5.1 功能增强
- 用户可以对任意选中的群组内部节点进行独立布局
- 保持群组边界的完整性
- 支持嵌套群组的逐层布局操作

### 5.2 用户体验
- 选择群组后可以独立进行内部布局
- 不影响其他区域的节点位置
- 视觉效果清晰，布局合理

## 6. 注意事项

### 6.1 性能考虑
- 群组内部节点数量可能很大，需要性能优化
- 边界约束计算可能需要多次迭代

### 6.2 边界情况
- 群组尺寸过小，无法容纳内部节点
- 群组内节点数量过多
- 嵌套结构过深（当前只处理单层）

该方案在保持现有架构的基础上，扩展了布局功能以支持嵌套节点的独立布局，同时考虑了边界约束和用户体验，为第二阶段开发提供了清晰的实现路径。
