# 数据模型与类型系统

## 核心模块详解

### 数据模型与类型系统 (src/types)

#### src/types/graph/models.ts
定义了整个应用的核心数据模型。

**核心类型**:

##### BlockEnum
节点类型枚举：
```typescript
enum BlockEnum {
  NODE = 'node',    // 普通节点
  GROUP = 'group'   // 群组节点
}
```

##### CommonNodeType
通用节点类型接口：
```typescript
interface CommonNodeType<T = any> {
  id: string;                          // 唯一标识符
  type: BlockEnum;                     // 节点类型
  position: { x: number; y: number };  // 位置坐标
  data?: T;                            // 自定义数据
  width?: number;                      // 宽度
  height?: number;                     // 高度
  selected?: boolean;                  // 是否选中
  dragging?: boolean;                  // 是否拖拽中
  parentId?: string;                   // 父节点 ID（用于嵌套）
}
```

##### Node（普通节点）
```typescript
interface Node extends CommonNodeType {
  type: BlockEnum.NODE;
  title: string;                       // 标题（必填）
  content?: string;                    // Markdown 内容
  attributes?: Record<string, any>;    // 结构化属性
  tags?: string[];                     // 标签列表
  summary?: string;                    // 摘要
  isEditing?: boolean;                 // 是否处于编辑状态
  groupId?: string;                    // 所属群组 ID
  validationError?: string;            // 验证错误信息
  isExpanded?: boolean;                // 是否展开
  customExpandedSize?: {               // 自定义展开尺寸
    width: number;
    height: number;
  };
  style?: CSSProperties;               // 样式
  createdAt: Date;                     // 创建时间
  updatedAt: Date;                     // 更新时间
}
```

**关键属性说明**:
- `isExpanded`: 控制节点的展开/收缩状态
- `customExpandedSize`: 保存用户手动调整的展开尺寸
- `groupId`: 如果节点属于某个群组，该字段指向群组 ID

##### Group（群组节点）
```typescript
interface Group extends CommonNodeType {
  type: BlockEnum.GROUP;
  title: string;                       // 群组标题
  content?: string;                    // 群组描述
  attributes?: Record<string, any>;    // 结构化属性
  tags?: string[];                     // 标签列表
  summary?: string;                    // 摘要
  isEditing?: boolean;                 // 是否处于编辑状态
  collapsed: boolean;                  // 是否收缩
  nodeIds: string[];                   // 包含的节点 ID 列表
  boundary: {                          // 边界范围
    minX: number;
    minY: number;
    maxX: number;
    maxY: number;
  };
  style?: CSSProperties;               // 样式
  createdAt: Date;                     // 创建时间
  updatedAt: Date;                     // 更新时间
}
```

**关键属性说明**:
- `nodeIds`: 维护群组内所有节点的 ID 列表
- `boundary`: 用于计算群组的实际显示边界
- `groupId`: 群组节点也可以嵌套在其他群组中

##### Edge（边/关系）
```typescript
interface Edge {
  id: string;                          // 唯一标识符
  source: string;                      // 源节点 ID
  target: string;                      // 目标节点 ID
  sourceHandle?: string;               // 源连接点
  targetHandle?: string;               // 目标连接点
  label?: string;                      // 标签
  groupId?: string;                    // 所属群组 ID
  style?: CSSProperties;               // 样式
  data?: {
    color?: string;                    // 颜色
    strokeWidth?: number;              // 线宽
    isCrossGroup?: boolean;            // 是否跨群组
    sourceGroupId?: string;            // 源节点群组 ID
    targetGroupId?: string;            // 目标节点群组 ID
    strokeDasharray?: string;          // 虚线样式
    weight?: number;                   // 权重
    strength?: number;                 // 强度
    direction?: 'unidirectional' | 'bidirectional' | 'undirected';
    customProperties?: Record<string, any>;
  };
  createdAt: Date;                     // 创建时间
  updatedAt: Date;                     // 更新时间
}
```

**关键属性说明**:
- `isCrossGroup`: 标识是否为跨群组关系
- `direction`: 关系的方向性（单向、双向、无向）
- `weight` 和 `strength`: 用于可视化不同强度的关系
- `groupId`: 边也可以属于某个群组

**Zod 验证 Schema**:
每个数据模型都有对应的 Zod Schema 用于运行时验证：
- `NodeSchema`: 验证普通节点数据
- `GroupSchema`: 验证群组节点数据
- `EdgeSchema`: 验证边数据

## 数据流与验证

### 输入验证
所有用户输入数据都通过 Zod Schema 进行验证：

```typescript
// 验证节点数据
const validatedNode = NodeSchema.parse(rawNodeData);

// 验证群组数据
const validatedGroup = GroupSchema.parse(rawGroupData);

// 验证边数据
const validatedEdge = EdgeSchema.parse(rawEdgeData);
```

### 数据同步
在布局系统中，节点数据在不同层级之间同步：

1. **Store层**: Zustand状态管理
2. **Layout层**: 布局算法处理
3. **ReactFlow层**: 可视化渲染

### 嵌套关系
支持多层嵌套关系：

**数据结构**:
```
GroupA (depth 0)
├─ NodeA1
├─ GroupB (depth 1)
│  ├─ NodeB1
│  ├─ NodeB2
│  └─ GroupC (depth 2)
│     └─ NodeC1
└─ NodeA2
```

**实现机制**:
- 通过 `groupId` 字段建立父子关系
- 在 `Node` 和 `Group` 接口中都有 `groupId` 属性
- 支持无限层级的嵌套（实际应用中建议限制最大深度）

## 与布局系统的集成

### 布局相关属性
节点和群组包含布局系统需要的特定属性：

**位置信息**:
- `position`: 节点在画布上的坐标
- `width`/`height`: 节点尺寸

**群组边界**:
- `boundary`: 群组的边界信息，用于布局约束
- `nodeIds`: 群组内节点列表，用于群组布局

**嵌套信息**:
- `groupId`: 节点所属群组，用于层级关系
- 嵌套群组也有 `groupId` 属性

### 布局优化
通过数据模型支持的属性，布局系统可以实现：

1. **群组内布局**: 利用 `groupId` 和 `nodeIds`
2. **碰撞检测**: 利用 `position`、`width`、`height`
3. **边优化**: 利用 `sourceHandle`、`targetHandle`
4. **层级布局**: 利用嵌套 `groupId` 关系

这种设计使得数据模型不仅支持基本的图编辑功能，还为复杂的布局算法提供了必要的数据支持。
