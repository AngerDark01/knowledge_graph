# 布局系统重构 - 执行流程详解

## 📋 文档说明

本文档详细说明三种布局场景的完整执行流程，基于真实源码编写。

**最后更新**：2025-11-24

---

## 🎯 三种典型布局场景

1. **画布整体布局**：用户点击"应用布局"按钮
2. **群组内部布局**：用户选中群组后点击"布局群组内部"
3. **递归全局布局**：用户点击"递归布局全部"按钮

每种场景使用不同的策略，但遵循统一的流程框架。

## 📐 场景1：画布整体布局

### 用户操作

```
用户操作：点击"应用布局"按钮
触发函数：LayoutControl.handleLayout()
```

### 完整执行流程

#### 第1步：UI层准备（LayoutControl.tsx）

**禁用交互**：
```
setIsProcessing(true)
setSelectedNodeId(null)
```

**获取数据**：
```
nodes = useGraphStore.getNodes()
edges = useGraphStore.getEdges()
```

**创建管理器**：
```
layoutManager = new LayoutManager()
```

**发起布局请求**：
```
layoutResult = await layoutManager.applyLayout(nodes, edges, {
  animate: true,
  useWeightedLayout: true
})
```

---

#### 第2步：智能策略选择（LayoutManager）

**选择逻辑**：
```
检查 options.strategy -> 无
检查 options.layoutMode -> 无
检查 options.targetGroupId -> 无
默认选择: 'canvas-layout'
```

**获取策略实例**：
```
strategy = this.strategies.get('canvas-layout')
// 返回 CanvasLayoutStrategy 实例
```

---

#### 第3步：策略执行 - 筛选顶层节点（CanvasLayoutStrategy）

**筛选条件**：
```
topLevelNodes = nodes.filter(node =>
  !node.groupId
)
```

**示例数据**：
```
输入节点（10个）：
  NodeA (无groupId)       ✅ 顶层
  NodeB (无groupId)       ✅ 顶层
  GroupX (无groupId)      ✅ 顶层
  NodeC (groupId: GroupX) ❌ 嵌套
  NodeD (groupId: GroupX) ❌ 嵌套
  GroupY (无groupId)      ✅ 顶层
  NodeE (groupId: GroupY) ❌ 嵌套
  NodeF (groupId: GroupY) ❌ 嵌套
  NodeG (groupId: GroupY) ❌ 嵌套
  NodeH (无groupId)       ✅ 顶层

筛选结果（5个）：
  [NodeA, NodeB, GroupX, GroupY, NodeH]
```

---

#### 第4步：计算节点权重

**权重计算流程**：

**4.1 统计边连接**：
```
for (edge of edges) {
  if (edge.source === node.id) outEdges++
  if (edge.target === node.id) inEdges++
}
totalEdges = inEdges + outEdges
```

**4.2 计算面积权重**：
```
area = node.width * node.height
areaWeight = min(1, area / 100000)
```

**4.3 计算边数权重**：
```
totalEdgeWeight = min(1, totalEdges / 10)
inEdgeWeight = min(1, inEdges / 10)
outEdgeWeight = min(1, outEdges / 10)
```

**4.4 综合权重**：
```
weight = areaWeight * 0.2
       + totalEdgeWeight * 0.3
       + inEdgeWeight * 0.25
       + outEdgeWeight * 0.25
```

**示例计算**：
```
NodeA:
  尺寸: 120x80 = 9600
  总边: 5条, 入边: 3条, 出边: 2条

  areaWeight = min(1, 9600/100000) = 0.096
  totalEdgeWeight = min(1, 5/10) = 0.5
  inEdgeWeight = min(1, 3/10) = 0.3
  outEdgeWeight = min(1, 2/10) = 0.2

  weight = 0.096*0.2 + 0.5*0.3 + 0.3*0.25 + 0.2*0.25
         = 0.294

权重排名（假设）：
  GroupY: 0.520 （最重要，中心位置）
  GroupX: 0.410
  NodeA:  0.294
  NodeB:  0.180
  NodeH:  0.120
```

---

#### 第5步：算法计算布局（GridCenterAlgorithm）

**5.1 权重排序**：
```
排序后顺序：
  [GroupY, GroupX, NodeA, NodeB, NodeH]
```

**5.2 计算网格参数**：
```
nodeCount = 5
optimalCols = ceil(sqrt(5)) = 3
rows = ceil(5 / 3) = 2

网格结构：
  Row 0: [GroupY, GroupX, NodeA]
  Row 1: [NodeB, NodeH]
```

**5.3 计算网格总尺寸**：
```
假设：
  avgWidth = 150
  avgHeight = 100
  horizontalSpacing = 150
  verticalSpacing = 100

gridWidth = 150*3 + 150*2 = 750
gridHeight = 100*2 + 100*1 = 300
```

**5.4 计算居中偏移**：
```
offsetX = -750 / 2 = -375
offsetY = -300 / 2 = -150
```

**5.5 计算每个节点位置**：
```
GroupY (row=0, col=0):
  x = -375 + 0*300 + 75 = -300
  y = -150 + 0*200 + 50 = -100
  位置: (-300, -100)

GroupX (row=0, col=1):
  x = -375 + 1*300 + 75 = 0
  y = -150 + 0*200 + 50 = -100
  位置: (0, -100)  // 中心位置

NodeA (row=0, col=2):
  x = -375 + 2*300 + 75 = 300
  y = -150 + 0*200 + 50 = -100
  位置: (300, -100)

NodeB (row=1, col=0):
  x = -375 + 0*300 + 75 = -300
  y = -150 + 1*200 + 50 = 100
  位置: (-300, 100)

NodeH (row=1, col=1):
  x = -375 + 1*300 + 75 = 0
  y = -150 + 1*200 + 50 = 100
  位置: (0, 100)
```

**布局效果**：
```
        (-300,-100)  (0,-100)    (300,-100)
          GroupY     GroupX       NodeA
                       ↑
                   最高权重
                   中心位置

        (-300,100)   (0,100)
          NodeB       NodeH
```

---

#### 第6步：碰撞检测与解决（CollisionResolver）

**6.1 检测碰撞**：
```
collisionCount = 0
for (i = 0; i < nodes.length; i++) {
  for (j = i+1; j < nodes.length; j++) {
    if (isColliding(nodes[i], nodes[j])) {
      collisionCount++
    }
  }
}
```

**6.2 迭代解决**：
```
假设检测到1个碰撞（NodeB和NodeH距离过近）

iteration = 0
while (iteration < 50 && hasCollisions()) {
  解决碰撞（推开节点）
  iteration++
}

解决后：
  NodeB: (-300, 100) -> (-320, 100)  向左推
  NodeH: (0, 100) -> (20, 100)       向右推
```

---

#### 第7步：更新嵌套节点位置

**7.1 识别嵌套节点**：
```
nestedNodes = nodes.filter(n => n.groupId)

结果：
  NodeC (groupId: GroupX)
  NodeD (groupId: GroupX)
  NodeE (groupId: GroupY)
  NodeF (groupId: GroupY)
  NodeG (groupId: GroupY)
```

**7.2 计算绝对位置**：

**NodeC（属于GroupX）**：
```
原始：
  GroupX: (200, 150)
  NodeC: (220, 170)  // 相对GroupX: (20, 20)

布局后：
  GroupX: (0, -100)  // 新位置

计算：
  relativeX = 220 - 200 = 20
  relativeY = 170 - 150 = 20

  newX = 0 + 20 = 20
  newY = -100 + 20 = -80

  NodeC新位置: (20, -80)
```

**NodeE（属于GroupY）**：
```
原始：
  GroupY: (500, 300)
  NodeE: (520, 320)  // 相对GroupY: (20, 20)

布局后：
  GroupY: (-300, -100)  // 新位置

计算：
  relativeX = 520 - 500 = 20
  relativeY = 320 - 300 = 20

  newX = -300 + 20 = -280
  newY = -100 + 20 = -80

  NodeE新位置: (-280, -80)
```

**7.3 多层嵌套处理**：
```
NestedNodePositionUpdater.updateNestedNodePositionsForCanvasLayout
处理多层嵌套结构，如 GroupA -> GroupB -> NodeX
确保所有层级的节点都更新到正确位置
```

**最终所有节点位置**：
```
顶层节点（5个）：
  GroupY: (-300, -100)
  GroupX: (0, -100)
  NodeA: (300, -100)
  NodeB: (-320, 100)
  NodeH: (20, 100)

嵌套节点（5个）：
  NodeC: (20, -80)      // 跟随GroupX
  NodeD: (50, -80)      // 跟随GroupX
  NodeE: (-280, -80)    // 跟随GroupY
  NodeF: (-250, -80)    // 跟随GroupY
  NodeG: (-280, -50)    // 跟随GroupY
```

---

#### 第8步：优化边连接点（EdgeOptimizer）

**8.1 遍历所有边**：
```
for (edge of edges) {
  sourceNode = 找到源节点
  targetNode = 找到目标节点

  bestHandles = 选择最优连接点(sourceNode, targetNode)

  edge.sourceHandle = bestHandles.source
  edge.targetHandle = bestHandles.target
}
```

**8.2 选择最优连接点**：
```
示例边: NodeA -> NodeB

NodeA位置: (300, -100)
NodeB位置: (-320, 100)

尝试16种组合：
  top->top, top->bottom, top->left, top->right
  bottom->top, bottom->bottom, ...
  ...

计算每种组合的距离，选择最短的：
  NodeA.left -> NodeB.right
  距离最短，选择这个组合

结果：
  edge.sourceHandle = 'left'
  edge.targetHandle = 'right'
```

---

#### 第9步：返回结果（CanvasLayoutStrategy）

**构建结果Map**：
```
nodePositions = new Map([
  ['GroupY', { x: -300, y: -100 }],
  ['GroupX', { x: 0, y: -100 }],
  ['NodeA', { x: 300, y: -100 }],
  ['NodeB', { x: -320, y: 100 }],
  ['NodeH', { x: 20, y: 100 }],
  ['NodeC', { x: 20, y: -80 }],
  ['NodeD', { x: 50, y: -80 }],
  ['NodeE', { x: -280, y: -80 }],
  ['NodeF', { x: -250, y: -80 }],
  ['NodeG', { x: -280, y: -50 }]
])

edgeHandles = new Map([
  ['edge1', { sourceHandle: 'left', targetHandle: 'right', ... }],
  ...
])

return {
  success: true,
  nodes: nodePositions,
  edges: edgeHandles,
  errors: [],
  stats: {
    duration: 125,
    iterations: 1,
    collisions: 1
  }
}
```

---

#### 第10步：UI更新（LayoutControl.tsx）

**10.1 启用布局模式**：
```
useGraphStore.getState().setIsLayoutMode(true)
```

**10.2 更新节点位置**：
```
for ([nodeId, position] of layoutResult.nodes) {
  updateNode(nodeId, { position })
}
```

**10.3 更新边连接点**：
```
for ([edgeId, edgeData] of layoutResult.edges) {
  updateEdge(edgeId, {
    sourceHandle: edgeData.sourceHandle,
    targetHandle: edgeData.targetHandle
  })
}
```

**10.4 额外边优化**：
```
延迟执行（确保节点位置已更新）：
  EdgeOptimizer.optimizeEdgeHandles(finalNodes, finalEdges)
  再次更新边连接点
```

**10.5 恢复交互**：
```
setIsLayoutMode(false)
setIsProcessing(false)
```

---

### 时间线总结

```

```

## 🔁 场景3：递归全局布局

### 用户操作

```
用户操作：点击"递归布局全部"按钮
触发函数：LayoutControl.handleRecursiveLayout()
```

### 完整执行流程

#### 第1步：构建嵌套层级树

**输入数据（假设）**：
```
节点结构：
  GroupA (depth 0)
    ├─ NodeA1 (depth 1)
    └─ GroupB (depth 1)
        ├─ NodeB1 (depth 2)
        └─ NodeB2 (depth 2)

  GroupC (depth 0)
    └─ NodeC1 (depth 1)

  NodeX (depth 0)
  NodeY (depth 0)
```

**构建深度分组**：
```
depthGroups = NestingTreeBuilder.groupNodesByDepth(nodes)

结果：
  Map {
    0 => [GroupA, GroupC, NodeX, NodeY],
    1 => [NodeA1, GroupB, NodeC1],
    2 => [NodeB1, NodeB2]
  }

maxDepth = 2
```

**日志输出**：
```
console.log('检测到 3 层嵌套结构，最大深度: 2')
```

---

#### 第2步：深度2布局（最深层）

**获取深度2的群组**：
```
groupsAtDepth2 = getGroupsAtDepth(depthGroups, 2)
// 结果: [] （深度2没有群组，只有节点）
```

**跳过**，因为没有群组需要布局。

---

#### 第3步：深度1布局

**获取深度1的群组**：
```
groupsAtDepth1 = getGroupsAtDepth(depthGroups, 1)
// 结果: [GroupB]
```

**布局GroupB**：

**3.1 检查子节点**：
```
children = [NodeB1, NodeB2]  // 2个子节点
```

## 🔄 通用流程模式

所有布局策略都遵循相同的通用流程：

```
1. 输入验证: 检查参数是否正确
2. 数据预处理: 筛选、排序、分组节点
3. 算法执行: 调用算法层计算位置
4. 工具处理: 碰撞解决、坐标转换、尺寸调整
5. 嵌套更新: 处理嵌套结构的节点
6. 边优化: 优化边的连接点
7. 结果构建: 生成LayoutResult
8. 返回结果: 异步返回给调用方
```

这种标准化的流程确保了所有策略的一致性，便于理解、扩展和维护。
