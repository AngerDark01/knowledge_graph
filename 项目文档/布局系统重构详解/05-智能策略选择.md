# 布局系统重构 - 智能策略选择机制

## 📋 文档说明

本文档详细说明 LayoutManager 的智能策略选择机制，基于真实源码编写。

**源码位置**：`src/services/layout/LayoutManager.ts`

**最后更新**：2025-11-24

## 🎯 策略选择的目标

### 设计理念

调用方**不需要了解策略的实现细节**，只需要：
1. 描述想要什么结果（通过 options 参数）
2. LayoutManager 自动选择最合适的策略

### 用户体验提升

**重构前**：
```typescript
// 用户需要知道策略名称
const layoutManager = new LayoutManager()
const strategy = new GridCenterLayoutStrategy()
layoutManager.registerStrategy('grid-center-layout', strategy)

await layoutManager.applyLayout(nodes, edges, {
  strategy: 'grid-center-layout',  // 手动指定
  targetGroupId: 'GroupX'           // 选项
})
```

**重构后**：
```typescript
// 用户只需描述需求
const layoutManager = new LayoutManager()

await layoutManager.applyLayout(nodes, edges, {
  targetGroupId: 'GroupX'  // 自动选择 group-layout
})
```

## 🧠 智能选择算法

### selectStrategy() 方法

**源码位置**：`src/services/layout/LayoutManager.ts` 中的 selectStrategy 方法

**方法签名**：
```typescript
private selectStrategy(options?: LayoutOptions): string
```

**输入**：
- `options`：布局选项对象

**输出**：
- `string`：策略ID（如 'canvas-layout'）

---

### 选择流程图

```
开始
  │
  ▼
是否有 options.strategy?
  │
  ├─ 是 ─────────────────────► 返回 options.strategy
  │
  └─ 否
      │
      ▼
是否有 options.layoutMode === 'recursive'?
      │
      ├─ 是 ─────────────────► 返回 'recursive-layout'
      │
      └─ 否
          │
          ▼
是否有 options.targetGroupId?
          │
          ├─ 是 ─────────────► 返回 'group-layout'
          │
          └─ 否 ──────────────► 返回 'canvas-layout' (默认)
```

---

### 源码实现

```typescript
private selectStrategy(options?: LayoutOptions): string {
  // 优先级1: 显式指定策略
  if (options?.strategy) {
    return options.strategy
  }

  // 优先级2: 递归布局模式
  if (options?.layoutMode === 'recursive') {
    return 'recursive-layout'
  }

  // 优先级3: 群组布局
  if (options?.targetGroupId) {
    return 'group-layout'
  }

  // 默认: 画布布局
  return 'canvas-layout'
}
```

## 📝 策略选择规则详解

### 规则1：显式指定策略（最高优先级）

**触发条件**：
```typescript
options.strategy !== undefined
```

**使用场景**：
- 用户明确知道要用哪个策略
- 测试特定策略
- 兼容旧代码

**示例**：
```typescript
// 示例1: 显式使用旧策略
await layoutManager.applyLayout(nodes, edges, {
  strategy: 'grid-center-layout'  // 使用旧策略
})
// 选择: grid-center-layout

// 示例2: 显式使用新策略
await layoutManager.applyLayout(nodes, edges, {
  strategy: 'canvas-layout'
})
// 选择: canvas-layout

// 示例3: 显式使用群组策略
await layoutManager.applyLayout(nodes, edges, {
  strategy: 'group-layout',
  targetGroupId: 'GroupX'
})
// 选择: group-layout
```

**优先级说明**：
- **最高优先级**
- 即使提供了其他选项（layoutMode、targetGroupId），也优先使用显式指定的策略
- 这保了向后兼容性

---

### 规则2：递归布局模式

**触发条件**：
```typescript
options.layoutMode === 'recursive'
```

**使用场景**：
- 需要布局所有嵌套层级
- 初始化整个图
- 重新布局所有节点

**示例**：
```typescript
await layoutManager.applyLayout(nodes, edges, {
  layoutMode: 'recursive',
  useWeightedLayout: true,
  onProgress: (progress) => {
    console.log(`进度: ${progress.processedNodes}/${progress.totalNodes}`)
  }
})
// 选择: recursive-layout
```

**为什么优先级高于 targetGroupId？**
```typescript
// 假设用户同时提供了两个选项
await layoutManager.applyLayout(nodes, edges, {
  layoutMode: 'recursive',
  targetGroupId: 'GroupX'  // 会被忽略
})
// 选择: recursive-layout（递归会处理所有群组，包括GroupX）
```

**设计考虑**：
- `layoutMode: 'recursive'` 表示**全局操作**
- `targetGroupId` 表示**局部操作**
- 全局优先级高于局部

---

### 规则3：群组布局

**触发条件**：
```typescript
options.targetGroupId !== undefined
```

**使用场景**：
- 用户选中特定群组
- 只需要布局群组内部
- 不影响其他节点

**示例**：
```typescript
await layoutManager.applyLayout(nodes, edges, {
  targetGroupId: 'GroupX',
  layoutScope: 'group',  // 可选，明确意图
  gridSpacing: 120
})
// 选择: group-layout
```

**检测逻辑**：
```typescript
// 只要有 targetGroupId 就触发
if (options?.targetGroupId) {
  return 'group-layout'
}

// 不检查 targetGroupId 的有效性
// 策略内部会验证群组是否存在
```

**为什么不验证群组存在？**
- 保持 selectStrategy 的单一职责（只负责选择）
- 验证由策略负责（GroupLayoutStrategy.applyLayout）
- 避免重复遍历节点列表

---

### 规则4：默认画布布局

**触发条件**：
```typescript
// 没有提供任何特殊选项
```

**使用场景**：
- 用户点击"应用布局"按钮
- 布局所有顶层节点
- 最常见的使用场景

**示例**：
```typescript
// 示例1: 最简单的调用
await layoutManager.applyLayout(nodes, edges)
// 选择: canvas-layout

// 示例2: 带权重布局
await layoutManager.applyLayout(nodes, edges, {
  useWeightedLayout: true
})
// 选择: canvas-layout

// 示例3: 自定义间距
await layoutManager.applyLayout(nodes, edges, {
  gridSpacing: 180
})
// 选择: canvas-layout
```

**为什么是默认？**
- 最常用的场景
- 处理范围适中（顶层节点）
- 性能开销合理

---

## 🎪 策略优先级总结

```
优先级顺序（从高到低）：

1. options.strategy（显式指定）
   ↓
2. options.layoutMode === 'recursive'（递归模式）
   ↓
3. options.targetGroupId（群组ID）
   ↓
4. 默认 'canvas-layout'
```

### 优先级表

| 选项组合 | 选择的策略 | 原因 |
|---------|-----------|------|
| `strategy: 'xxx'` | `xxx` | 显式指定，最高优先级 |
| `layoutMode: 'recursive'` | `recursive-layout` | 递归模式 |
| `layoutMode: 'recursive'` + `targetGroupId: 'X'` | `recursive-layout` | 递归优先 |
| `targetGroupId: 'X'` | `group-layout` | 群组布局 |
| `useWeightedLayout: true` | `canvas-layout` | 默认 |
| 无选项 | `canvas-layout` | 默认 |

## 🔄 applyLayout() 完整流程

### 方法签名

```typescript
async applyLayout(
  nodes: (Node | Group)[],
  edges: Edge[],
  options?: LayoutOptions
): Promise<LayoutResult>
```

### 执行步骤

#### 步骤1：智能选择策略

```typescript
const strategyId = this.selectStrategy(options)
```

#### 步骤2：获取策略实例

```typescript
const strategy = this.strategies.get(strategyId)

if (!strategy) {
  throw new Error(`策略 ${strategyId} 未注册`)
}
```

**容错处理**：
```typescript
// 如果策略不存在，抛出错误
// 这通常不会发生，因为 registerDefaultStrategies() 已注册所有策略
```

#### 步骤3：调用策略执行布局

```typescript
const result = await strategy.applyLayout(nodes, edges, options)
```

**日志输出**：
```typescript
console.log(`LayoutManager: 使用策略 ${strategyId}`)
console.log(`LayoutManager: 布局完成，耗时 ${result.stats.duration}ms`)
```

#### 步骤4：返回结果

```typescript
return result
```

## 🔧 策略注册机制

### registerDefaultStrategies() 方法

**调用时机**：
```typescript
constructor() {
  this.strategies = new Map()
  this.registerDefaultStrategies()
}
```

**注册流程**：

#### 1. 创建共享实例

```typescript
const collisionResolver = new CollisionResolver()
const edgeOptimizer = new EdgeOptimizer()
const nestingTreeBuilder = new NestingTreeBuilder()
```

**为什么共享？**
- 减少内存开销
- 这些工具类是无状态的
- 可以安全地被多个策略共享

#### 2. 创建策略实例（依赖注入）

```typescript
// 群组布局策略
const groupStrategy = new GroupLayoutStrategy(
  gridAlgorithm,           // 注入算法
  collisionResolver,       // 注入工具
  nestedNodePositionUpdater // 注入工具
)

// 画布布局策略
const canvasStrategy = new CanvasLayoutStrategy(
  gridCenterAlgorithm,     // 注入算法
  collisionResolver,       // 注入工具
  edgeOptimizer,          // 注入工具
  nestedNodePositionUpdater // 注入工具
)

// 递归布局策略
const recursiveStrategy = new RecursiveLayoutStrategy(
  groupStrategy,           // 注入其他策略
  canvasStrategy,          // 注入其他策略
  nestingTreeBuilder,      // 注入工具
  edgeOptimizer,          // 注入工具
  nestedNodePositionUpdater // 注入工具
)
```

#### 3. 注册策略

```typescript
this.strategies.set('group-layout', groupStrategy)
this.strategies.set('canvas-layout', canvasStrategy)
this.strategies.set('recursive-layout', recursiveStrategy)

console.log('✅ 策略已注册: Group Layout (group-layout)')
console.log('✅ 策略已注册: Canvas Layout (canvas-layout)')
console.log('✅ 策略已注册: Recursive Layout (recursive-layout)')
```

#### 4. 异步加载旧策略（向后兼容）

```typescript
import('./strategies/GridCenterLayoutStrategy')
  .then(({ GridCenterLayoutStrategy }) => {
    const oldStrategy = new GridCenterLayoutStrategy()
    this.strategies.set('grid-center-layout', oldStrategy)
    console.log('✅ 旧策略已注册（向后兼容）: Grid Center Layout')
  })
  .catch(error => {
    console.error('加载旧策略失败:', error)
  })
```

**为什么异步加载旧策略？**
- 减少初始加载时间
- 旧策略不是必需的（新代码不应使用）
- 只有显式指定时才需要

## 🎯 设计优势

### 1. 简化调用

**重构前**：
```typescript
// 调用方需要了解3个概念
const manager = new LayoutManager()              // 1. 管理器
const strategy = new GridCenterLayoutStrategy()  // 2. 策略类
manager.registerStrategy('grid-center-layout', strategy)  // 3. 策略ID

await manager.applyLayout(nodes, edges, {
  strategy: 'grid-center-layout'
})
```

**重构后**：
```typescript
// 调用方只需了解1个概念
const manager = new LayoutManager()  // 1. 管理器

await manager.applyLayout(nodes, edges, {
  targetGroupId: 'GroupX'  // 描述需求即可
})
```

### 2. 降低错误率

**可能的错误（重构前）**：
```typescript
// 错误1: 拼写错误
await manager.applyLayout(nodes, edges, {
  strategy: 'grid-centr-layout'  // 少了一个e
})

// 错误2: 策略ID不匹配
manager.registerStrategy('my-strategy', strategy)
await manager.applyLayout(nodes, edges, {
  strategy: 'grid-center-layout'  // 使用了错误的ID
})

// 错误3: 忘记注册
await manager.applyLayout(nodes, edges, {
  strategy: 'some-strategy'  // 策略未注册
})
```

**重构后**：
```typescript
// 不需要知道策略ID，降低出错可能
await manager.applyLayout(nodes, edges, {
  targetGroupId: 'GroupX'
})
```

### 3. 易于扩展

**添加新策略**：
```typescript
// 1. 实现新策略
class TreeLayoutStrategy implements ILayoutStrategy {
  // ...
}

// 2. 在 registerDefaultStrategies 中注册
const treeStrategy = new TreeLayoutStrategy()
this.strategies.set('tree-layout', treeStrategy)

// 3. 添加选择规则到 selectStrategy
private selectStrategy(options?: LayoutOptions): string {
  if (options?.layoutType === 'tree') {
    return 'tree-layout'
  }
  // ... 其他规则
}

// 4. 使用新策略
await manager.applyLayout(nodes, edges, {
  layoutType: 'tree'
})
```

### 4. 向后兼容

**支持旧代码**：
```typescript
// 旧代码可以继续工作
await manager.applyLayout(nodes, edges, {
  strategy: 'grid-center-layout'
})

// 同时支持新代码
await manager.applyLayout(nodes, edges, {
  targetGroupId: 'GroupX'
})
```

## 📚 最佳实践

### 推荐用法

**✅ 画布布局**：
```typescript
await manager.applyLayout(nodes, edges)
// 或
await manager.applyLayout(nodes, edges, {
  useWeightedLayout: true
})
```

**✅ 群组布局**：
```typescript
await manager.applyLayout(nodes, edges, {
  targetGroupId: selectedGroupId
})
```

**✅ 递归布局**：
```typescript
await manager.applyLayout(nodes, edges, {
  layoutMode: 'recursive',
  onProgress: (progress) => updateUI(progress)
})
```

### 避免的用法

**❌ 冲突选项**：
```typescript
// 不推荐：layoutMode 和 targetGroupId 冲突

  layoutMode: 'recursive',
  targetGroupId: 'GroupX'  // 会被忽略
})
```

**❌ 不必要的显式指定**：
```typescript
// 不推荐：智能选择已经够用

  strategy: 'group-layout',
  targetGroupId: 'GroupX'
})
```
