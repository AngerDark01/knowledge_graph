# 布局系统重构详解

## 1. 重构背景

在知识图谱项目中，原有的布局系统存在以下问题：
- 代码结构混乱，职责不清晰
- 不同布局策略混杂在一起，难以维护和扩展
- 群组布局和画布布局逻辑耦合严重
- 缺乏统一的布局接口和策略管理机制

为了解决这些问题，我们进行了全面的重构，采用了策略模式和模块化设计。

## 2. 重构目标

- 实现职责分离，每个模块专注于单一功能
- 引入策略模式，支持多种布局算法
- 提供统一的布局接口，便于使用和扩展
- 改善系统性能，特别是碰撞检测和群组边界更新
- 提高代码可读性和可维护性

## 3. 架构设计

### 3.1 核心架构模式

重构后的布局系统采用了以下设计模式：

1. **策略模式 (Strategy Pattern)**: 为布局算法提供统一接口
2. **工厂模式 (Factory Pattern)**: 根据参数自动选择合适的布局策略
3. **单一职责原则 (SRP)**: 每个类和模块专注单一功能
4. **依赖注入 (Dependency Injection)**: 组件之间通过接口解耦

## 4. 详细设计

### 4.1 布局管理器 (LayoutManager)

**文件路径**: `/src/services/layout/LayoutManager.ts`

LayoutManager 是布局系统的核心管理器，负责：

- **策略注册**: 统一管理所有布局策略
- **策略选择**: 根据参数智能选择最合适的布局策略
- **流程控制**: 控制布局执行流程，处理并发和取消操作

```ts
export class LayoutManager implements ILayoutManager {
  // 策略注册机制
  private strategies: Map<string, ILayoutStrategy>;
  
  // 智能选择策略
  private selectStrategy(options?: LayoutOptions): string {
    if (options?.targetGroupId) return 'group-layout';    // 群组内部布局
    if (options?.layoutMode === 'recursive') return 'recursive-layout'; // 递归布局
    return 'canvas-layout'; // 默认画布布局
  }
  
  // 统一布局入口
  async applyLayout(nodes: (Node | Group)[], edges: Edge[], options?: LayoutOptions): Promise<LayoutResult>
}
```

### 4.2 布局策略实现

#### 4.2.1 群组布局策略 (GroupLayoutStrategy)

**文件路径**: `/src/services/layout/strategies/GroupLayoutStrategy.ts`

专门处理群组内节点布局的策略，主要功能：

1. **筛选子节点**: 获取目标群组的直接子节点
2. **相对布局计算**: 使用算法计算相对于群组的节点位置
3. **坐标转换**: 将相对坐标转换为绝对坐标
4. **碰撞解决**: 处理节点间碰撞
5. **群组调整**: 自适应调整群组尺寸

```ts
export class GroupLayoutStrategy implements ILayoutStrategy {
  async applyLayout(
    nodes: (Node | Group)[],
    edges: Edge[],
    options?: LayoutOptions
  ): Promise<LayoutResult> {
    // 1. 获取目标群组
    const parentGroup = this.findGroup(nodes, options?.targetGroupId);
    
    // 2. 获取直接子节点
    const children = this.getDirectChildren(nodes, options?.targetGroupId);
    
    // 3. 使用算法计算相对布局
    const relativeLayout = this.algorithm.calculate(children, options);
    
    // 4. 转换为绝对坐标
    const absoluteLayout = CoordinateTransformer.relativeToAbsoluteForGroup(
      relativeLayout,
      parentGroup
    );
    
    // 5. 碰撞解决
    const resolvedLayout = this.collisionResolver.resolve(absoluteLayout);
    
    // 6. 群组尺寸调整
    const adjustedNodes = GroupSizeAdjuster.adjustSingleGroup(
      options?.targetGroupId,
      nodes
    );
    
    return {/* 布局结果 */};
  }
}
```

#### 4.2.2 画布布局策略 (CanvasLayoutStrategy)

**文件路径**: `/src/services/layout/strategies/CanvasLayoutStrategy.ts`

处理整个画布上节点的布局，通常使用中心网格算法。

#### 4.2.3 递归布局策略 (RecursiveLayoutStrategy)

**文件路径**: `/src/services/layout/strategies/RecursiveLayoutStrategy.ts`

处理多层嵌套群组结构的递归布局，从最深层开始向上逐层布局。

### 4.3 布局算法

#### 4.3.1 网格算法 (GridAlgorithm)

**文件路径**: `/src/services/layout/algorithms/GridAlgorithm.ts`

从左上角开始的网格布局算法，适用于群组内部布局：

- 从(0,0)开始布局，所有坐标为正数
- 节点按行优先顺序排列
- 自动计算最优行列数

#### 4.3.2 网格中心算法 (GridCenterAlgorithm)

**文件路径**: `/src/services/layout/algorithms/GridCenterAlgorithm.ts`

从画布中心开始的网格布局算法，适用于全画布布局。

### 4.4 工具类

#### 4.4.1 坐标转换工具 (CoordinateTransformer)

**文件路径**: `/src/services/layout/utils/CoordinateTransformer.ts`

提供各种坐标系统之间的转换功能：

- 相对坐标转绝对坐标
- 第四象限转换（确保所有坐标为正）
- 计算群组锚点

#### 4.4.2 群组尺寸调整工具 (GroupSizeAdjuster)

**文件路径**: `/src/services/layout/utils/GroupSizeAdjuster.ts`

根据子节点的实际占用空间自动调整父群组的尺寸：

- 计算子节点相对于父群组的边界
- 计算需要的内容区域大小
- 应用最小尺寸限制
- 批量调整多个群组的大小

#### 4.4.3 碰撞解决工具 (CollisionResolver)

**文件路径**: `/src/services/layout/utils/CollisionResolver.ts`

使用迭代式排斥力算法解决节点重叠问题：

- 检测节点间碰撞
- 使用最小平移向量分离重叠节点
- 控制最大迭代次数避免无限循环

## 5. 按钮交互流程

### 5.1 群组内部布局按钮

**文件路径**: `/src/components/graph/controls/LayoutControl.tsx`

当用户点击"布局群组内部"按钮时：

1. **UI触发**: 按钮点击事件触发 `handleGroupLayout` 函数
2. **参数验证**: 检查是否选中了群组，且群组包含子节点
3. **策略调用**: 创建 `LayoutManager` 实例，调用 `applyLayout` 方法
4. **选项设置**: 传入 `{ targetGroupId: selectedNodeId }` 参数
5. **策略选择**: `LayoutManager` 智能选择 `GroupLayoutStrategy`
6. **布局计算**: 策略执行布局算法，计算节点新位置
7. **状态更新**: 更新 Store 中的节点位置和群组尺寸
8. **边界更新**: 调用 `updateGroupBoundary` 更新群组边界
9. **边优化**: 执行额外的边连接点优化

```tsx
// 群组内部布局按钮定义
{isGroupSelected && childrenCount > 0 && (
  <Button
    variant="outline"

    onClick={handleGroupLayout}  // ← 触发函数
    title={`布局选中群组的 ${childrenCount} 个子节点`}
  >
    <ArrowsPointingOutIcon className="w-4 h-4" />
    布局群组内部 ({childrenCount})
  </Button>
)}
```

## 7. 扩展性

### 7.1 新增布局策略

要添加新的布局策略，只需：

1. 实现 `ILayoutStrategy` 接口
2. 在 `LayoutManager` 中注册策略
3. 在 `selectStrategy` 方法中添加选择逻辑

### 7.2 配置扩展

所有配置项都集中管理，易于扩展和修改。

## 8. 总结

通过重构，我们实现了：

- **职责分离**: 每个模块专注单一功能
- **策略模式**: 支持多种布局算法
- **可扩展性**: 易于添加新的布局策略
- **性能优化**: 防抖、缓存等优化措施
- **代码质量**: 清晰的架构和良好的可维护性

重构后的布局系统结构清晰、性能优异、易于扩展，为知识图谱编辑器提供了强大而灵活的布局功能。
