# 布局系统重构 - 算法层详解

## 📋 文档说明

本文档详细说明算法层的设计和实现，基于真实源码编写。

**源码位置**：`src/services/layout/algorithms/`

**最后更新**：2025-11-21

---

## 🎯 算法层职责

算法层负责纯粹的位置计算，具有以下特点：

1. **无业务逻辑**：只计算位置，不关心节点类型、群组关系等
2. **可插拔**：实现统一接口，可以随时替换算法
3. **可配置**：通过选项参数调整算法行为
4. **可复用**：同一个算法可以被多个策略使用

---

## 🧩 算法接口设计

### ILayoutAlgorithm 接口

**文件**：`src/services/layout/algorithms/ILayoutAlgorithm.ts` (72行)

**接口定义**：

```typescript
interface ILayoutAlgorithm {
  readonly name: string       // 算法名称
  readonly id: string         // 算法唯一标识

  calculate(
    nodes,                   // 输入节点列表
    options?                 // 可选配置
  ): (Node | Group)[]        // 返回布局后的节点

  validateOptions?(options): boolean  // 可选：验证配置
}
```

**设计理念**：

1. **只读属性**：name 和 id 不可修改，确保算法身份稳定
2. **统一方法**：所有算法都实现 calculate() 方法
3. **灵活配置**：通过 options 传递算法参数
4. **可选验证**：validateOptions() 用于验证配置有效性

**LayoutAlgorithmOptions 类型**：

```typescript
interface LayoutAlgorithmOptions {
  horizontalSpacing?: number      // 水平间距
  verticalSpacing?: number        // 垂直间距
  useWeightedOrder?: boolean      // 是否使用权重排序
  nodeWeights?: Map<string, number>  // 节点权重
  minSpacing?: number             // 最小间距
  maxColumns?: number             // 最大列数
}
```

## 1️⃣ GridAlgorithm - 网格布局算法

### 核心特点

**文件**：`src/services/layout/algorithms/GridAlgorithm.ts` (166行)

**定位**：从左上角 (0,0) 开始的网格布局，用于**群组内部布局**

**坐标系**：第四象限坐标系（所有坐标都是正数）

### 算法流程

#### 阶段1：准备工作

**输入验证**：
- 检查节点列表是否为空
- 空列表直接返回

**获取间距配置**：
```
horizontalSpacing = options?.horizontalSpacing || 默认值(150)
verticalSpacing = options?.verticalSpacing || 默认值(100)
minSpacing = 20px（最小间距）
```

**获取节点尺寸**：
```
对于每个节点：
  width = node.width || 默认宽度(120)
  height = node.height || 默认高度(80)

计算平均尺寸：
  avgWidth = sum(widths) / count
  avgHeight = sum(heights) / count
```

---

#### 阶段2：计算网格参数

**计算最优列数**：

```
方法：基于节点数量的平方根

optimalCols = Math.ceil(sqrt(nodeCount))

例如：
- 9个节点 -> 3列
- 10个节点 -> 4列
- 16个节点 -> 4列
```

**为什么使用平方根？**
- 让网格接近正方形，视觉效果更好
- 避免过长或过宽的布局

**计算行数**：
```
rows = Math.ceil(nodeCount / cols)

例如：
- 10个节点，4列 -> 3行
```

**限制列数**：
```
if (options.maxColumns) {
  cols = Math.min(cols, maxColumns)
}
```

---

#### 阶段3：逐个计算节点位置

**网格坐标计算**：
```
对于第 i 个节点：
  row = Math.floor(i / cols)   // 所在行
  col = i % cols                // 所在列
```

**位置计算公式**：
```
x = col * (avgWidth + horizontalSpacing) + nodeWidth / 2
y = row * (avgHeight + verticalSpacing) + nodeHeight / 2
```

**公式解释**：
- `col * (avgWidth + horizontalSpacing)`：列偏移量
- `+ nodeWidth / 2`：节点中心点偏移

**示例计算**：
```
假设：
- avgWidth = 120
- avgHeight = 80
- horizontalSpacing = 150
- verticalSpacing = 100

第0个节点（row=0, col=0）：
  x = 0 * (120 + 150) + 60 = 60
  y = 0 * (80 + 100) + 40 = 40
  位置：(60, 40)

第1个节点（row=0, col=1）：
  x = 1 * (120 + 150) + 60 = 330
  y = 0 * (80 + 100) + 40 = 40
  位置：(330, 40)

第3个节点（row=1, col=0）：
  x = 0 * (120 + 150) + 60 = 60
  y = 1 * (80 + 100) + 40 = 220
  位置：(60, 220)
```

---

#### 阶段4：确保第四象限坐标

**调用工具转换**：
```
使用 CoordinateTransformer.toQuadrantFour()
确保所有坐标 >= 0
```

**为什么需要？**
- 虽然公式计算的坐标已经是正数
- 但为了确保健壮性，统一转换一次

---

### 适用场景

**主要用途**：
- ✅ 群组内部布局（GroupLayoutStrategy）
- ✅ 需要从固定点开始的布局
- ❌ 不适合需要居中的场景

**优势**：
- 简单直观
- 计算高效
- 适合群组内的相对布局

**限制**：
- 不支持居中
- 不考虑节点权重

## 2️⃣ GridCenterAlgorithm - 居中网格算法

### 核心特点

**文件**：`src/services/layout/algorithms/GridCenterAlgorithm.ts` (177行)

**定位**：以原点为中心的网格布局，用于**画布顶层布局**

**坐标系**：可能包含负坐标的笛卡尔坐标系

### 算法流程

#### 阶段1：准备和排序

**权重排序（可选）**：

```
if (useWeightedOrder && nodeWeights) {
  nodes = 按权重降序排序(nodes)
}
```

**权重排序的意义**：
- 权重高的节点排在中心位置
- 重要节点更靠近原点
- 视觉焦点更突出

**获取配置**：
```
horizontalSpacing = options?.horizontalSpacing || 150
verticalSpacing = options?.verticalSpacing || 100
```

---

#### 阶段2：计算网格参数

**与 GridAlgorithm 相同**：
```
optimalCols = Math.ceil(sqrt(nodeCount))
rows = Math.ceil(nodeCount / cols)
```

---

#### 阶段3：计算网格总尺寸

**网格占据的总空间**：
```
gridWidth = avgWidth * cols + horizontalSpacing * (cols - 1)
gridHeight = avgHeight * rows + verticalSpacing * (rows - 1)
```

**示例**：
```
假设：
- 3列 2行
- avgWidth = 120
- avgHeight = 80
- horizontalSpacing = 150
- verticalSpacing = 100

gridWidth = 120 * 3 + 150 * 2 = 660
gridHeight = 80 * 2 + 100 * 1 = 260
```

---

#### 阶段4：计算起始偏移（居中关键）

**目标**：让整个网格以原点为中心

**偏移量计算**：
```
offsetX = -gridWidth / 2
offsetY = -gridHeight / 2
```

**示例**：
```
gridWidth = 660
gridHeight = 260

offsetX = -660 / 2 = -330
offsetY = -260 / 2 = -130
```

**效果**：
```
网格左上角位于：(-330, -130)
网格右下角位于：(330, 130)
网格中心位于：(0, 0)
```

---

#### 阶段5：逐个计算节点位置

**位置计算公式**：
```
x = offsetX + col * (avgWidth + horizontalSpacing) + nodeWidth / 2
y = offsetY + row * (avgHeight + verticalSpacing) + nodeHeight / 2
```

**公式解释**：
- `offsetX`：整体向左平移，实现居中
- `col * (avgWidth + horizontalSpacing)`：列偏移
- `+ nodeWidth / 2`：节点中心点偏移

**示例计算**：
```
offsetX = -330
offsetY = -130

第0个节点（row=0, col=0）：
  x = -330 + 0 * 270 + 60 = -270
  y = -130 + 0 * 180 + 40 = -90
  位置：(-270, -90)

第1个节点（row=0, col=1）：
  x = -330 + 1 * 270 + 60 = 0
  y = -130 + 0 * 180 + 40 = -90
  位置：(0, -90)

第2个节点（row=0, col=2）：
  x = -330 + 2 * 270 + 60 = 270
  y = -130 + 0 * 180 + 40 = -90
  位置：(270, -90)
```

**观察**：中间列的节点 x=0，实现了水平居中

---

### 权重排序详解

**权重的作用**：
- 决定节点在网格中的顺序
- 高权重节点排在前面（靠近中心）

**排序规则**：
```
nodes.sort((a, b) => {
  const weightA = nodeWeights.get(a.id) || 0
  const weightB = nodeWeights.get(b.id) || 0
  return weightB - weightA  // 降序
})
```

**权重计算（在 CanvasLayoutStrategy 中）**：
```
权重 = 面积权重 * 0.2
      + 总边数权重 * 0.3
      + 入度权重 * 0.25
      + 出度权重 * 0.25
```

**布局效果**：
```
假设3x2网格，权重从高到低：[A, B, C, D, E, F]

网格布局：
  A    B    C
  D    E    F

中心位置：B（第0行第1列）
A、C 在第一行左右两侧
D、E、F 在第二行
```

---

### 适用场景

**主要用途**：
- ✅ 画布顶层布局（CanvasLayoutStrategy）
- ✅ 需要居中显示的场景
- ✅ 需要突出重要节点的场景

**优势**：
- 视觉居中效果好
- 支持权重排序
- 重要节点更突出

**限制**：
- 可能产生负坐标（需要画布支持）
- 计算稍复杂

## 3️⃣ EdgeOptimizer - 边优化算法

### 核心特点

**文件**：`src/services/layout/algorithms/EdgeOptimizer.ts`

**定位**：优化边的连接点位置，减少边的交叉和重叠

### 算法职责

**输入**：
- 节点列表（已布局）
- 边列表

**输出**：
- 优化后的边列表（包含最佳连接点）

### 优化策略

#### 1. 连接点选择

**4个可选位置**：
- `top`：节点上边中点
- `bottom`：节点下边中点
- `left`：节点左边中点
- `right`：节点右边中点

#### 2. 最优连接点计算

**目标**：选择让边最短且最直的连接点组合

**方法**：
```
对于每条边：
  尝试 source 的4个位置 x target 的4个位置 = 16种组合
  计算每种组合的总长度
  选择长度最短的组合
```

#### 3. 避免交叉

**检测交叉**：
- 计算边与其他边的交叉情况
- 调整连接点以减少交叉

**优先级**：
- 长度最短 > 避免交叉
- 实际应用中权衡两者

---

### 使用场景

**应用于**：
- CanvasLayoutStrategy 布局后
- RecursiveLayoutStrategy 布局后

**效果**：
- 边更整洁
- 减少视觉混乱
- 提升图的可读性

| **起点** | (0, 0) | 居中原点 | N/A |

## 🎨 算法选择指南

### 何时使用 GridAlgorithm？

**适用于**：
- 群组内部布局
- 需要从固定点开始
- 不需要居中效果
- 性能要求高

**GroupLayoutStrategy 使用它**：
```
群组内布局流程：
1. GridAlgorithm 计算相对位置（从0,0开始）
2. CoordinateTransformer 转换为绝对坐标
```

### 何时使用 GridCenterAlgorithm？

**适用于**：
- 画布顶层布局
- 需要居中显示
- 有节点权重信息
- 需要突出重要节点

**CanvasLayoutStrategy 使用它**：
```
画布布局流程：
1. 计算节点权重
2. GridCenterAlgorithm 计算居中位置
3. 返回可能包含负坐标的结果
```

### 何时使用 EdgeOptimizer？

**适用于**：
- 所有布局完成后
- 需要优化边的显示
- 减少视觉混乱

**所有策略都使用它**：
```
布局完成后：
EdgeOptimizer.optimizeEdgeHandles(nodes, edges)
```

## 📐 坐标系设计思想

### GridAlgorithm 的第四象限坐标系

**为什么全正？**
1. 群组内部使用相对坐标
2. 相对坐标不应该有负数
3. 后续转换为绝对坐标更简单

**流程**：
```
GridAlgorithm 输出相对坐标（全正）
    ↓
CoordinateTransformer 转换为绝对坐标
    ↓
在画布上渲染
```

### GridCenterAlgorithm 的笛卡尔坐标系

**为什么可以有负数？**
1. 画布顶层布局需要居中效果
2. 居中意味着原点在中心
3. 自然会有负坐标

**流程**：
```
GridCenterAlgorithm 输出居中坐标（可负）
    ↓
直接在画布上渲染（画布支持负坐标）
```

## 💡 扩展建议

### 如何添加新算法？

**步骤**：

1. **创建算法类**：
```typescript
export class NewAlgorithm implements ILayoutAlgorithm {
  readonly name = 'New Algorithm'
  readonly id = 'new-algorithm'

  calculate(nodes, options): (Node | Group)[] {
    // 实现算法逻辑
  }
}
```

2. **在策略中使用**：
```typescript
const strategy = new SomeStrategy(
  new NewAlgorithm(),  // 注入新算法
  ...
)
```

3. **测试算法**：
- 编写单元测试
- 验证各种场景

### 可能的新算法

**力导向布局**：
- 基于物理模拟
- 节点之间有引力/斥力
- 自动找到平衡位置

**圆形布局**：
- 节点排列成圆形
- 适合展示循环关系

**层次布局**：
- 按层级排列节点
- 适合树形结构
