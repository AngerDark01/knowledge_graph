# 知识图谱编辑器项目本体结构（规划版）

> **说明**：本文档基于现有本体结构，规划了多画布管理、用户管理和本地存储功能的实现。
>
> **标注说明**：
> - ✅ 已实现
> - 🚧 规划中（待实现）
> - 📝 需要修改的现有文件

---

## 1. 项目概述
这是一个基于 Next.js 和 React Flow 的知识图谱编辑器，支持嵌套群组、节点转换、自动布局等功能。

**新增功能**：
- 多画布管理（主画布/子画布，支持嵌套）
- 用户管理（预留设计，当前默认0号用户）
- 本地存储持久化（localStorage/IndexedDB）
- 左右侧边栏布局（可调整宽度）

---

## 3. 主要数据流向

### 3.1 现有数据流（图编辑器）✅
1. **应用启动**: `src/app/page.tsx` → `GraphPageContent.tsx` → 初始化状态
2. **节点生命周期**: 创建 → 添加到 store → 渲染到 ReactFlow → 交互 → 更新状态
3. **布局处理**: API 调用 → LayoutManager → 布局策略 → ELK 转换 → 位置更新
4. **状态同步**: Zustand store ↔ UI 组件 ↔ ReactFlow 组件

### 3.2 新增数据流（工作空间管理）🚧
1. **应用启动加载**:
   ```
   src/app/page.tsx → WorkspaceLayout →
   StorageManager.load() → Zustand workspace store →
   LeftSidebar + GraphPageContent + RightSidebar
   ```

2. **画布切换流程**:
   ```
   用户点击画布 → CanvasTreeItem →
   workspaceStore.switchCanvas(id) →
   graphStore.loadCanvasData(id) →
   StorageManager.loadCanvas(id) →
   GraphPageContent 重新渲染
   ```

3. **画布创建流程**:
   ```
   用户创建画布 → CanvasTree.createCanvas() →
   workspaceStore.createCanvas() →
   StorageManager.saveCanvas() →
   CanvasTree 更新
   ```

4. **自动保存流程**:
   ```
   图数据变更 → graphStore 更新 →
   persistenceMiddleware 监听 →
   StorageManager.saveCanvas(currentCanvasId) →
   localStorage/IndexedDB
   ```

5. **画布删除流程**:
   ```
   用户删除画布 → CanvasTreeItem.delete() →
   workspaceStore.deleteCanvas(id) →
   StorageManager.deleteCanvas(id) →
   切换到默认画布
   ```

## 4. 架构模式

### 4.1 现有架构✅
- **状态管理**: 使用 Zustand 进行全局状态管理，采用切片模式
- **UI 框架**: React + React Flow 实现图可视化
- **后端服务**: Next.js App Router + API Routes
- **类型安全**: 使用 Zod 进行数据验证，TypeScript 保证类型安全
- **布局算法**: ELK.js 用于自动布局
- **样式**: Tailwind CSS + 自定义样式

### 4.2 新增架构模式🚧
- **本地存储**: 适配器模式（StorageAdapter 接口 + LocalStorage/IndexedDB 实现）
- **持久化**: Zustand 中间件自动同步状态到本地存储
- **数据迁移**: 版本化数据格式 + 迁移策略
- **UI 布局**: 三栏布局（左侧边栏 + 中间画布 + 右侧边栏），支持可调整宽度
- **画布管理**: 树形结构（类似文件系统，支持嵌套）

## 7. 数据结构设计

### 7.1 工作空间数据结构🚧

```typescript
// src/types/workspace/models.ts

// 用户模型（预留设计）
interface User {
  id: string // 默认 "user_0"
  name: string
  avatar?: string
  createdAt: string
}

// 画布模型
interface Canvas {
  id: string // 唯一标识
  name: string // 画布名称
  parentId: string | null // 父画布ID，null 表示根画布
  children: string[] // 子画布ID列表
  graphData: {
    nodes: Node[] // 节点数据
    edges: Edge[] // 边数据
  }
  viewportState: {
    x: number
    y: number
    zoom: number
  }
  isCollapsed: boolean // 是否折叠子画布
  createdAt: string
  updatedAt: string
}

// 工作空间模型
interface Workspace {
  userId: string // 当前用户ID
  currentCanvasId: string // 当前选中的画布ID
  canvases: Canvas[] // 所有画布列表
  canvasTree: CanvasTree // 画布树形结构
}

// 画布树节点
interface CanvasTreeNode {
  id: string
  name: string
  isCollapsed: boolean
  children: CanvasTreeNode[]
}

type CanvasTree = CanvasTreeNode[]
```

## 8. 开发规范要点

1. **模块化设计**：每个功能模块独立，避免代码堆砌

3. **状态管理**：使用 Zustand 切片模式，保持状态逻辑清晰
4. **组件设计**：遵循单一职责原则，组件粒度适中

6. **性能优化**：大数据量场景考虑虚拟滚动、懒加载
7. **可扩展性**：预留数据库接口，方便未来升级
8. **代码复用**：共性逻辑抽取为 hooks 和工具函数
