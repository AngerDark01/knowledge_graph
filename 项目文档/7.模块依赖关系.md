#### 4. 跨层依赖关系

**高内聚低耦合设计**:
- 算法层不依赖策略层，只依赖数据模型
- 工具层不依赖策略层，可独立使用
- 策略层组合算法层和工具层实现业务逻辑
- 管理层协调各策略层并提供统一接口

**依赖倒置原则应用**:
- 高层模块（LayoutManager）不直接依赖低层模块
- 两者都依赖抽象（ILayoutStrategy, ILayoutAlgorithm）
- 策略层依赖注入算法层和工具层实例

#### 5. 运行时依赖关系

**布局执行路径**:
```
用户操作 (UI) → LayoutControl → LayoutManager → 策略 → 算法/工具 → Store更新 → UI渲染
```

**数据流向**:
```
Store (nodes, edges) → 布局算法 → 布局结果 → Store更新 → UI同步
```

这种架构设计确保了模块间的松耦合，便于独立测试和扩展，同时保持了功能的完整性。
