# 知识图谱编辑器项目本体结构

## 1. 项目概述
这是一个基于 Next.js 和 React Flow 的知识图谱编辑器，支持嵌套群组、节点转换、自动布局等功能。

## 3. 主要数据流向
1. **应用启动**: `src/app/page.tsx` → `GraphPageContent.tsx` → 初始化状态
2. **节点生命周期**: 创建 → 添加到 store → 渲染到 ReactFlow → 交互 → 更新状态
3. **布局处理**: API 调用 → LayoutManager → 布局策略 → ELK 转换 → 位置更新
4. **状态同步**: Zustand store ↔ UI 组件 ↔ ReactFlow 组件

## 4. 架构模式
- **状态管理**: 使用 Zustand 进行全局状态管理，采用切片模式
- **UI 框架**: React + React Flow 实现图可视化
- **后端服务**: Next.js App Router + API Routes
- **类型安全**: 使用 Zod 进行数据验证，TypeScript 保证类型安全
- **布局算法**: ELK.js 用于自动布局
- **样式**: Tailwind CSS + 自定义样式

## 5. 关键依赖关系
- 项目核心依赖于 React Flow 进行图可视化
- Zustand 用于状态管理，是项目数据流的核心
- ELK.js 用于自动布局，是项目图布局的关键
- Next.js 作为主要框架，处理路由和 API
- Zod 确保数据验证和类型安全
