# 节点与 Domain 产品交互规格

日期：2026-05-06

## 1. 文档定位

这份文档从用户使用角度描述 Ontology Canvas 的节点和 Domain 交互逻辑。它不是代码实现说明，而是后续开发、验收和 UI 设计的产品契约。

技术实现可以调整，但最终体验必须满足这里的规则。

## 2. 产品目标

Ontology Canvas 不是普通白板。用户在画布上操作的对象不是随意卡片，而是有语义结构的本体元素：

- Class / Concept / Function / Component 等本体节点。
- Domain / 上下文边界 / 模块边界。
- 节点内部属性、规则、约束、接口、行为等结构化内容。
- 节点之间的语义关系。

用户应能直接在画布上完成“建模、分组、移动、查看、编辑、折叠、导航”，而不需要理解内部数据结构。

## 2.1 与新版 PRD 的关系

`prd.md` 已把长期产品形态收敛为“节点即容器 / 节点内部是子画布”。因此本文件中的 Domain 不再理解为长期独立实体，而是当前实现阶段用于表达“容器、上下文边界、内部空间”的过渡概念。

`UI_REFERENCE_INTERACTION_SPEC_2026-05-07.zh-CN.md` 已把四张 UI 参考图抽象为节点嵌套、正交边、LOD 大图和子画布导航的长期规格。本文继续负责短期节点与 Domain / 容器交互验收；涉及视觉状态、LOD、正交边、MiniMap、Breadcrumb 和子画布导航时，以 UI 参考规格和新版路线图为补充执行依据。

短期规则：

- 当前代码仍保留 Domain / Group 作为可见容器，用于修复嵌套、拖拽、resize、折叠和性能问题。
- Domain 交互必须稳定，因为它会迁移成“容器节点 / 子画布入口”的基础能力。
- 后续产品语义应逐步从“独立 Domain”迁移到“任意本体节点都可拥有内部画布空间”。

长期规则：

- Class / Concept / Module / Service 等节点都可以成为容器节点。
- 双击或入口动作进入节点内部子画布。
- Breadcrumb、MiniMap、搜索定位、跨画布引用跳转围绕子画布体系设计。
- Domain 只作为一种特殊容器视图或分类，不作为唯一分组机制。

## 3. 基础交互原则

### 3.1 交互要稳定

用户拖动、创建、编辑后，元素不能因为点击、选中、缩放、保存或重新渲染而跳回旧位置。

验收标准：

- 拖动节点后，点击该节点不回弹。
- 拖动节点后，点击画布空白处不回弹。
- 缩放、平移画布后，节点位置不变化。
- 保存再加载后，节点仍在用户最后放置的位置。

### 3.2 交互要顺滑

拖动过程中不能明显卡顿、抖动、闪烁或频繁跳位。

验收标准：

- 拖动 100 个以内的节点画布时，主交互应保持连续。
- 拖动 Domain 时，内部节点应跟随移动，不出现延迟明显的二次跳动。
- 拖拽期间不做全图保存、全图布局、全图边路径重算。
- 缩放远景时，节点 UI 自动降级显示，避免完整属性 DOM 全量渲染。

### 3.3 行为优先于实现

用户只关心：

- 节点是否在正确的 Domain 内。
- Domain 是否能包住子节点。
- 父 Domain 移动时子节点是否跟随。
- 节点属性是否能在节点上看见和编辑。

实现可以是 ReactFlow、Canvas 或其他渲染方案，但不能改变这些行为。

## 4. Domain 交互

### 4.1 在 Domain 内创建节点

当用户选中一个 Domain 并点击“新增节点”时，新节点应默认出现在该 Domain 内部。

系统应自动完成：

1. 新节点出现在 Domain 可见区域内。
2. 新节点和 Domain 建立父子关系。
3. Domain 尺寸能容纳新节点。
4. 如果已有多个子节点，新节点应放在合理的空位，避免完全重叠。

验收标准：

- 用户不需要手动指定父子关系。
- 新节点不会出现在 Domain 外面。
- 新节点创建后拖动父 Domain，会跟随父 Domain 一起移动。

### 4.2 归入 Domain / 容器

第一版不再优先采用“拖到 Domain 内就自动改归属”的隐式策略。用户选中节点后，应在控制面板或右侧 Inspector 中看到当前层级可以归入的 Domain / 容器，点击后显式归入。

系统应自动完成：

1. 建立节点和目标 Domain / 容器的父子关系。
2. 将节点移动到目标容器内部的合理位置。
3. 必要时扩展目标容器尺寸。
4. 如果原来属于其他容器，应同步移除旧父子关系。

验收标准：

- 普通拖动不悄悄改变节点归属。
- 归属变更必须来自明确操作。
- 点击归入后，节点不会出现在容器外部。
- 归入后拖动父容器，节点跟随移动。

### 4.3 从 Domain 移出节点

当用户把节点拖出 Domain 边界时，产品需要明确策略。当前建议第一版采用“限制在 Domain 内”的策略，避免父子关系意外丢失。

建议规则：

- 已归属 Domain 的节点默认不能被拖出父 Domain。
- 如果未来需要移出，应提供明确操作，例如右键“移出 Domain”或属性面板中修改所属 Domain。

验收标准：

- 普通拖动不会悄悄破坏父子关系。
- 用户能通过显式操作改变归属。

### 4.4 Domain 自动扩展

当用户在 Domain 内创建、移动、展开或调整节点尺寸，导致内容接近或超过边界时，Domain 应自动扩展。

验收标准：

- 子节点不会被 Domain 裁掉。
- Domain 自动扩展后，用户保存再加载仍保持扩展后的大小。
- 多层 Domain 中，内层 Domain 扩展后，外层 Domain 也能必要地跟随扩展。

### 4.5 移动父 Domain

当用户拖动 Domain 时，Domain 内所有子节点和子 Domain 必须保持相对位置一起移动。

验收标准：

- 移动父 Domain 后，内部结构看起来像一个整体被搬走。
- 子节点之间的相对距离不变。
- 子 Domain 与其内部节点的相对关系不变。
- 移动后点击任何子节点，不会跳回移动前位置。

### 4.6 Domain 折叠

Domain 应支持折叠。折叠后不显示内部完整节点，只显示摘要。

建议摘要内容：

- Domain 名称。
- 子节点数量。
- 子 Domain 数量。
- 关系数量。
- 可选状态标记，例如有警告、有未完成字段。

验收标准：

- 折叠后画布复杂度明显降低。
- 展开后内部节点恢复原位置。
- 折叠状态可保存。

## 5. 本体节点交互

### 5.1 节点不是普通卡片

本体节点应类似 UML 类图节点，至少能在节点上直接表达：

- 标题。
- 本体类型，例如 Class / Concept / Function / Component。
- 属性列表。
- 字段类型或值。
- 约束、规则、接口、行为等分区。
- 可选标签、状态或校验提示。

用户不应必须打开右侧面板才能知道节点大概是什么。

### 5.2 节点上直接查看属性

节点默认应能展示关键属性。属性多时可以分层显示。

建议显示层级：

```text
完整模式：标题、类型、主要属性、字段类型、状态
紧凑模式：标题、类型、属性数量、状态
轮廓模式：标题或缩写、类型色条
点状模式：最小标记
```

验收标准：

- 近距离能看清节点属性。
- 远距离不会因为属性太多导致卡顿。
- 缩放时信息密度逐级降低，而不是所有 DOM 一直完整显示。

### 5.3 节点上新增属性

用户应能从节点自身发起新增属性操作，而不只能去右侧编辑栏。

建议交互：

- 节点 hover 或选中时出现新增属性入口。
- 新增属性后在节点内部出现新行。
- 属性可快速编辑名称、值、数据类型。
- 复杂字段再进入右侧编辑栏完整编辑。

验收标准：

- 用户能在画布上快速补字段。
- 新增字段不会导致节点布局错乱。
- 属性新增后 Domain 能必要地扩展。

### 5.4 节点属性折叠

节点属性可能很多，因此节点本身应支持属性折叠。

建议规则：

- 单个分区可折叠，例如 Fields / Constraints / Interfaces。
- 整个节点可切换展开/收起。
- 收起后保留属性数量和异常状态。

验收标准：

- 属性多的节点不会把画布撑爆。
- 折叠状态可保存。
- 折叠节点仍能参与布局和连线。

### 5.5 节点编辑栏

右侧编辑栏用于完整编辑，不替代节点上的快速查看和轻量编辑。

编辑栏负责：

- 完整标题、类型、描述。
- 属性列表批量编辑。
- 字段类型、默认值、约束。
- 所属 Domain。
- 子图链接。
- 元数据和高级配置。

验收标准：

- 右侧编辑栏修改后，节点 UI 同步更新。
- 节点上快速编辑后，右侧编辑栏同步更新。
- 两者不出现互相覆盖或保存顺序冲突。

## 6. 节点 UI 配置规则

节点 UI 必须可配置，不能把尺寸、字号、间距、圆角、色彩硬编码在组件内部。

### 6.1 配置和样式分离

配置控制结构密度：

- 节点默认宽高。
- Domain 默认宽高。
- 节点 padding。
- 属性行高度。
- 标题高度。
- 字号层级。
- 分区间距。
- 折叠阈值。
- LOD 阈值。

样式控制视觉表达：

- 语义颜色。
- 边框。
- 阴影。
- hover / selected / invalid 状态。
- 动画和过渡。

验收标准：

- 调整节点默认宽度，只改配置文件。
- 调整属性行间距，只改配置文件或 token。
- 换主题颜色，只改语义 token。
- 改节点 UI，不需要改交互 model。

### 6.2 推荐配置位置

建议后续落位：

```text
features/ontology-canvas/config/nodeViewTokens.ts
features/ontology-canvas/config/domainViewTokens.ts
features/ontology-canvas/config/canvasInteractionConfig.ts
```

注意：

- UI token 不得被布局算法直接消费。
- 布局算法需要尺寸时，由 adapter 生成 Layout DTO。
- 算法配置和 UI token 分开维护。

## 7. 前端分层要求

节点相关能力必须拆开，避免改 UI 破坏交互逻辑。

目标结构：

```text
features/ontology-canvas/
  model/
    document/
    interactions/
    inspector/
    layout/
  ui/
    ClassNodeView.tsx
    DomainNodeView.tsx
    NodeFieldList.tsx
    NodeSection.tsx
  blocks/
    OntologyCanvasBlock.tsx
    NodeInspectorBlock.tsx
  adapters/
    react-flow/
    legacy-graph/
  config/
    nodeViewTokens.ts
    domainViewTokens.ts
```

分层规则：

- `model` 只处理数据、命令、交互规则，不 import UI。
- `ui` 只收 props 和 tokens，不直接碰 store、接口、ReactFlow。
- `blocks` 负责接线，从 model/store 拿数据，交给 ui。
- `adapters` 负责渲染引擎转换。
- `config` 提供尺寸、间距、密度、LOD 阈值等配置。

验收标准：

- 改节点样式，不改拖拽逻辑。
- 改 Domain 嵌套规则，不改节点 UI。
- 改右侧编辑栏，不改画布投影 adapter。
- 改 ReactFlow 适配，不改本体 model。

## 8. 性能体验要求

用户感知目标：

- 500 节点内可以编辑。
- 2000 节点可以浏览。
- 更大的图通过 Domain 折叠、Subgraph 拆分、LOD 和视口裁剪处理。

实现经验与参考案例已沉淀到 `CANVAS_INTERACTION_PERFORMANCE_RESEARCH_2026-05-06.zh-CN.md`。产品验收仍以本节为准；研究文档用于约束后续代码实现不要在拖拽中保存、布局、全图重算或破坏 Domain 嵌套事务。

交互性能规则：

- 拖拽中不保存。
- 拖拽中不自动布局。
- 拖拽中不全量重算边。
- resize 中不写旧 graph store，不触发额外渲染后副作用。
- resize 结束后才提交本体 view 尺寸，并按需级联容器边界。
- 节点移动或 resize 结束后，相关边锚点可增量重算，并写回本体 `edgeViews`，不能只写临时显示缓存。
- 缩放中按 LOD 降低节点 DOM。
- 普通节点远景时外框尺寸也应按 LOD 降级，不能只缩小内部文字和图标。
- LOD 显示尺寸只属于 adapter 投影，不覆盖本体 view 保存的真实节点尺寸。
- 缩小后再放大，节点必须恢复近景外框尺寸；LOD 自动测量事件不能被当作用户 resize 保存。
- 屏幕外节点不渲染或降级渲染。
- Domain 折叠后内部节点不完整渲染。

验收标准：

- 拖动父 Domain 时内部节点跟随，不发生明显卡顿。
- 缩放到远景时，不再渲染完整属性列表。
- 展开大 Domain 时才恢复内部节点细节。

## 9. 后续开发验收顺序

后续阶段以 `DEVELOPMENT_ROADMAP_2026-05-07.zh-CN.md` 为准。本文只定义节点、Domain / 容器、UI 和交互验收契约。

第一优先级：修复交互正确性。

1. Domain 内创建节点，自动建立父子关系。
2. Domain 自动扩展，保证子节点在内部。
3. 拖动父 Domain，子节点保持相对位置移动。
4. 节点移动后不回弹。
5. Domain 折叠/展开保留内部位置。

第二优先级：节点 UI 产品化。

1. 本体节点显示标题、类型、属性。
2. 节点上可新增属性。
3. 属性支持分区和折叠。
4. 节点 UI 使用统一配置和 token。
5. LOD 下节点逐级降级显示。

第三优先级：编辑栏和 UI 解耦。

1. 节点 UI、节点交互 model、右侧编辑栏分离。
2. 快速编辑和完整编辑状态一致。
3. 旧 graph store 不再作为节点位置和属性真相源。

第四优先级：正交边与语义锚点。

1. 当前贝塞尔边迁移为正交边。
2. Top / Right / Bottom / Left 通用锚点稳定。
3. Field / Method / Section 语义锚点有数据结构和 UI 表达。
4. 边样式、线型、箭头、端点符号进入配置和 token。
5. 远景 LOD 下边标签和多边关系可降级或聚合。

第五优先级：子画布导航。

1. 任意本体节点可以拥有内部子画布。
2. 进入 / 返回 / Breadcrumb / MiniMap 可用。
3. 全局 Inspector 不重复生成。
4. 子画布视口、展开状态和导航历史可保存。

明确后置：

- 布局算法 DTO、worker、patch history 放在交互/UI/正交边/子画布之后。
- PostgreSQL adapter 放在 JSON 本体文档和 workspace repository 稳定之后。
- 导入/导出协议重新设计，不恢复 Mermaid 主线。

## 10. 当前检查结论

对照以上产品规格，当前项目状态如下：

| 项目 | 状态 | 说明 |
|------|------|------|
| Domain 内创建节点 | 部分满足 | 新增入口已本体化；普通节点显式归入容器已在 Inspector 第一批落地，Domain 内新增节点的空位策略仍需浏览器验证 |
| Domain 自动父子关系 | 基础满足 | 语义层有 `domainId/nodeIds`，拖拽归属不再隐式改；普通节点可在 Inspector 直接点击归入 / 移出容器，并同步本体父子关系和旧 display cache |
| 父 Domain 移动带动子节点 | 基础满足 | 已通过本体 interaction patch 同步平移后代；仍需浏览器大图验证 |
| 节点移动不回弹 | 基础满足 | 节点拖拽结束写本体 view，再投影旧 display cache；仍需浏览器回归验证 |
| 节点显示本体属性 | 阶段满足 | `ClassNodeView` 已展示类型、字段、方法、规则、接口、关系、子画布摘要和 LOD；完整/紧凑/轮廓/点状形态、hover/selected/readonly/disabled 状态已具备 Phase B 版本 |
| 节点上新增属性 | 基础满足 | 节点顶栏可新增普通 Field，各 Fields / Methods / Rules / Interfaces 分区标题可新增对应分类字段，并写入本体 `fields`；完整 LOD 下字段名、字段值、dataType、字段分类切换、删除和当前分区内上移/下移已支持；复杂批量编辑仍待补 |
| 节点属性折叠 | 阶段满足 | Fields / Methods / Rules / Interfaces / Relationships 分区已可折叠，折叠状态保存到本体 `nodeViews.collapsedSections`；折叠后显示分区摘要，字段过多时以 `+N hidden` 降级 |
| UI 配置化 | 阶段满足 | 节点和 Domain UI token 已进入 feature config，字段行编辑、分区摘要、状态 rail、内部空间入口尺寸已 token 化；边和子画布导航 token 放入 Phase D/E |
| UI/交互/编辑栏隔离 | 阶段满足 | 节点 UI、Domain UI、字段列表、分区、Inspector UI 已进入 `features/ontology-canvas/ui`，wrapper 只负责接 store/document；旧 graph store 清理放入 Phase F |
| 显式归入 / 移出容器 | 基础满足 | 普通本体节点在 Inspector Container 区显示可归入容器按钮；点击后写本体 `domainId/nodeIds`，自动放入容器内部空位并按需扩展容器；Domain 自身归入父容器、多层候选过滤和浏览器大图验证仍待补 |
| Domain 折叠投影 | 基础满足 | 折叠容器仍显示自身摘要，内部后代节点不再进入 ReactFlow 投影，连到隐藏后代或缺失端点的边不渲染；摘要边、跨容器关系聚合和浏览器大图验证仍待 Phase C/新 UI 交互稳定后续批次 |
| LOD 真实降 DOM | 部分满足 | 节点 UI 已按 `dot/outline/compact/full` 降级，普通节点外框也已按 LOD 显示尺寸缩小，并阻止 LOD 自动尺寸测量写入真实尺寸；折叠 Domain 已能隐藏内部后代 DOM，Domain 远景外框缩小、边 LOD 和聚合视图仍未完成 |
| resize 顺滑性 | 部分满足 | resize 结束写本体 view，自定义展开尺寸已迁入本体 view；仍需浏览器性能验证和减少节点 DOM |
| 边锚点自适应 | 部分满足 | 拖拽/resize 后相关边锚点已写入本体 `edgeViews`；正交边和语义锚点尚未完成 |
