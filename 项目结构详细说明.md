# 知识图谱应用详细项目结构说明

## 项目概述
这是一个基于ReactFlow构建的知识图谱可视化编辑器，支持节点创建、群组管理、节点拖拽、连接等操作。项目采用Zustand作为状态管理，实现了复杂的状态同步机制，特别是处理了ReactFlow的onNodesChange回调与Zustand状态管理之间的同步问题。

## 核心模块详细说明

### 1. 状态管理模块 (stores/graph/)

#### index.ts
此文件创建全局的图形状态store，整合了所有状态切片：
- 定义GraphStore类型，联合所有状态切片的类型
- 使用Zustand的create函数创建全局store
- 通过set和get函数将所有切片的状态和方法合并到单一store中
- 导出useGraphStore Hook供组件使用

#### nodes/ 子模块
将节点操作分为多个关注点分离的文件：

**basicOperations.ts**
基础节点操作切片，实现以下功能：

**状态属性：**
- nodes: (Node | Group)[] - 存储所有节点和群组
- selectedNodeId: string | null - 当前选中的节点ID
- selectedEdgeId: string | null - 当前选中的边ID

**方法功能：**

1. addNode: 添加新节点或群组到状态
   - 将节点/群组添加到nodes数组中
   - 保持节点的完整结构（包含position、data、style等属性）
   - 根据节点类型设置默认尺寸（NoteNode: 350x280, Group: 300x200）
   - 如果节点属于群组，约束其位置在群组边界内
   - 添加历史记录快照

2. updateNode: 更新节点或群组的特定属性
   - 实现深度合并，确保不丢失未更新的属性
   - 包含验证逻辑（如title不能为空）
   - 保留原有属性如position、data、style等
   - 更新updatedAt时间戳
   - 自动同步width/height到style属性
   - 如果节点属于群组，确保其位置在群组边界内
   - 根据更新类型决定是否添加历史记录快照

3. deleteNode: 从状态中删除节点
   - 从nodes数组中过滤掉指定ID的节点
   - 添加历史记录快照

4. getNodes & getNodeById: 获取节点的辅助方法

5. setSelectedNodeId & setSelectedEdgeId: 更新选中状态

**groupOperations.ts**
群组操作切片，实现以下功能：

1. addGroup: 添加群组
   - 与addNode类似，但专用于群组类型

2. updateGroup: 更新群组属性
   - 特殊处理群组内节点ID列表变更
   - 当节点ID列表更新时，自动更新相关节点的groupId

3. deleteGroup: 删除群组
   - 从群组中移除节点的groupId引用
   - 从nodes数组中移除群组本身

4. addNodeToGroup: 将节点添加到群组
   - 更新节点的groupId属性
   - 添加节点ID到群组的nodeIds列表
   - 处理节点从原群组移除的逻辑

5. removeNodeFromGroup: 从群组中移除节点
   - 删除节点的groupId属性
   - 从群组的nodeIds列表中移除节点ID

**groupBoundaryOperations.ts**
群组边界操作切片，实现以下功能：

1. updateGroupBoundary: 根据群组内节点自动调整群组边界
   - 计算群组内所有节点的边界
   - 自动调整群组的位置和尺寸以包含所有节点
   - 添加内边距确保节点不会紧贴边界

**constraintOperations.ts**
约束操作切片，实现以下功能：

1. constrainNodeToGroupBoundary: 将节点位置约束在群组边界内
   - 检查节点是否在指定群组边界内
   - 如果超出边界，返回约束后的位置

**types.ts**
定义节点操作切片的类型和辅助函数：

1. safePosition: 验证并修复位置坐标
2. safeNumber: 验证并修复数值（如width、height）
3. NodeOperationsSlice: 定义节点操作切片的类型

#### edgesSlice.ts
边状态管理切片，实现以下功能：

**状态属性：**
- edges: Edge[] - 存储所有边

**方法功能：**

1. addEdge: 添加新边
   - 将边添加到edges数组中
   - 包含source、target、label等基本属性

2. updateEdge: 更新边的特定属性
   - 实现部分更新，保留未修改的属性
   - 更新updatedAt时间戳

3. deleteEdge: 从状态中删除边
   - 从edges数组中过滤掉指定ID的边

4. getEdges & getEdgeById: 获取边的辅助方法

#### canvasViewSlice.ts
画布视图状态管理切片，实现以下功能：

**状态属性：**
- viewport: ViewportState - 视口状态，包含x, y, zoom
- canvasSize: { width: number; height: number } - 画布尺寸

**方法功能：**

1. setViewport: 设置整个视口状态
   - 替换当前视口位置和缩放

2. updateViewport: 部分更新视口状态
   - 只更新指定的视口属性

3. setCanvasSize: 设置画布尺寸

#### historySlice.ts
历史记录状态管理切片，实现以下功能：

**状态属性：**
- history: Array<{ nodes: Node[]; edges: Edge[] }> - 历史记录快照数组
- historyIndex: number - 当前历史记录索引
- maxSize: number - 最大历史记录数量（默认50）

**方法功能：**

1. addHistorySnapshot: 添加当前状态到历史记录
   - 保存当前nodes和edges的快照
   - 截断当前索引后的历史记录
   - 如果超过最大大小，删除最早的记录

2. canUndo: 检查是否可以撤销
   - 当前索引大于0时返回true

3. canRedo: 检查是否可以重做
   - 当前索引小于历史记录长度-1时返回true

4. undo: 撤销操作
   - 恢复到上一个历史记录快照

5. redo: 重做操作
   - 恢复到下一个历史记录快照

6. clearHistory: 清空历史记录

### 2. 图形组件模块 (components/graph/)

#### GraphPageContent.tsx
应用主入口组件，负责整合所有图形编辑功能：

**状态管理：**
- 使用useNodesState和useEdgesState管理ReactFlow状态
- 与useGraphStore同步图形数据

**渲染功能：**
- 节点类型：NoteNode（普通节点）和GroupNode（群组节点）
- 边类型：CustomEdge（自定义边）
- ReactFlow组件配置：缩放、连接模式、选择行为等
- 控制组件：MiniMap、Controls、Background

**事件处理：**
- onNodesChange: 核心状态同步逻辑
  - 处理节点选择变化
  - 处理节点删除
  - 处理尺寸调整（防抖机制）
    - 使用resizeTimeoutRef管理防抖定时器
    - 只在resizing = false时更新store
    - 100ms防抖延迟避免频繁更新
  - 处理位置变化（拖拽）
    - 检测dragging: false事件（拖拽结束）
    - 使用reactFlowInstance.getNode获取最终位置
    - 同步更新群组和群组内节点的位置
  - 群组边界自动更新

- onEdgesChange: 处理边的交互（选择、删除）
- onNodeDragStop: 处理节点拖拽停止
  - 使用restrictNodePositionToGroup限制节点在群组边界内
- onConnect: 处理节点连接
- onNodeClick, onEdgeClick: 处理选择事件
- onDragOver, onDrop: 处理拖放添加新节点

**组件结构：**
- 侧边栏：节点/边编辑器、工具栏
- 主画布：ReactFlow组件

**依赖的Hooks：**
- useNodeHandling: 节点添加和管理
- useEdgeHandling: 边的添加和管理
- useKeyboardShortcuts: 快捷键处理
- useSelectionHandling: 选择状态管理
- useViewportControls: 视口控制

### 3. 类型定义 (types/graph/models.ts)

**BlockEnum**
枚举类型，定义节点类型：
- NODE: 普通节点
- GROUP: 群组节点

**CommonNodeType<T>**
基础节点类型：
- id: string - 节点唯一标识
- type: BlockEnum - 节点类型
- position: { x: number; y: number } - 节点在画布上的位置
- data?: T - 节点自定义数据
- width?: number - 节点宽度
- height?: number - 节点高度
- selected?: boolean - 是否被选中
- dragging?: boolean - 是否正在拖拽
- parentId?: string - 父节点ID（用于嵌套）
- [key: string]: any - 任意额外属性

**Node**
普通节点类型，继承CommonNodeType：
- type: BlockEnum.NODE - 固定为NODE类型
- title: string - 节点标题

## 核心技术机制

### 1. 状态同步机制
onNodesChange是ReactFlow与Zustand状态管理之间的关键桥梁，处理三种主要事件：

1. **位置变化**：拖拽节点后，在dragging: false事件中获取节点最终位置
2. **尺寸变化**：拖拽调整节点大小时，防抖处理以避免频繁更新
3. **选择/删除**：同步UI选择状态到store状态

### 2. 群组管理机制
- 群组内节点边界约束：节点不能被拖出群组边界
- 群组边界自动扩展：根据内部节点自动调整群组尺寸
- 群组移动同步：移动群组时同步移动内部所有节点

### 3. 性能优化策略
- React.memo避免不必要的重渲染
- useCallback优化回调函数
- 防抖处理频繁更新操作
- 深度合并避免状态丢失

## 设计原则和模式

1. **单一职责原则**
   - 节点组件只负责渲染
   - ReactFlow负责UI交互
   - onNodesChange是唯一的状态同步桥梁
   - Store只负责数据存储

2. **明确保留原则**
   - 更新节点时，明确保留所有未修改的属性
   - 使用深度合并确保不丢失任何属性

3. **防抖延迟原则**
   - 只在操作完成时更新store
   - 使用防抖避免频繁状态更新
   - UI先响应，状态后更新

## 模块依赖关系

1. **GraphPageContent.tsx** 依赖于:
   - ReactFlow: 提供图形编辑界面和交互功能
   - useGraphStore: 同步全局状态
   - 各种hooks: useNodeHandling, useEdgeHandling, useKeyboardShortcuts, useSelectionHandling, useViewportControls
   - 节点组件: NoteNode, GroupNode
   - 边组件: CustomEdge

2. **状态管理模块** 依赖关系:
   - index.ts 合并所有切片 (nodes, edges, canvasView, history)
   - nodes/* 模块内部相互协作处理节点操作
   - Node操作会触发historySlice中的addHistorySnapshot

3. **节点组件** 依赖关系:
   - BaseNode: 基础渲染组件，被NoteNode和GroupNode继承
   - NoteNode: 依赖MarkdownRenderer渲染内容，依赖BaseNode提供基础功能
   - GroupNode: 依赖BaseNode提供基础功能，处理群组特定逻辑

4. **工具函数** 依赖关系:
   - validation.ts 提供验证功能，被节点编辑器和内容编辑器使用
   - 各组件根据需要调用验证函数确保数据有效性

5. **类型定义** 依赖关系:
   - models.ts 定义了所有核心类型，被整个项目使用
   - Zod Schemas 用于运行时验证，确保数据结构正确性
