# 知识图谱应用详细项目结构说明

## 项目概述
这是一个基于ReactFlow构建的知识图谱可视化编辑器，支持节点创建、群组管理、节点拖拽、连接等操作。项目采用Zustand作为状态管理，实现了复杂的状态同步机制，特别是处理了ReactFlow的onNodesChange回调与Zustand状态管理之间的同步问题。

## 核心模块详细说明

### 1. 状态管理模块 (stores/graph/)

#### index.ts
此文件创建全局的图形状态store，整合了所有状态切片：
- 定义GraphStore类型，联合所有状态切片的类型
- 使用Zustand的create函数创建全局store
- 通过set和get函数将所有切片的状态和方法合并到单一store中
- 导出useGraphStore Hook供组件使用

#### nodesSlice.ts
节点和群组状态管理切片，实现以下功能：

**状态属性：**
- nodes: (Node | Group)[] - 存储所有节点和群组
- selectedNodeId: string | null - 当前选中的节点ID
- selectedEdgeId: string | null - 当前选中的边ID

**方法功能：**

1. addNode: 添加新节点或群组到状态
   - 将节点/群组添加到nodes数组中
   - 保持节点的完整结构（包含position、data、style等属性）

2. updateNode: 更新节点或群组的特定属性
   - 实现深度合并，确保不丢失未更新的属性
   - 包含验证逻辑（如title不能为空）
   - 保留原有属性如position、data、style等
   - 更新updatedAt时间戳

3. deleteNode: 从状态中删除节点
   - 从nodes数组中过滤掉指定ID的节点

4. getNodes & getNodeById: 获取节点的辅助方法

5. setSelectedNodeId & setSelectedEdgeId: 更新选中状态

6. addGroup: 添加群组（与addNode类似但专用于群组）

7. updateGroup: 更新群组属性
   - 特殊处理群组内节点ID列表变更
   - 当节点ID列表更新时，自动更新相关节点的groupId

8. deleteGroup: 删除群组
   - 从群组中移除节点的groupId引用
   - 从nodes数组中移除群组本身

9. addNodeToGroup: 将节点添加到群组
   - 更新节点的groupId属性
   - 添加节点ID到群组的nodeIds列表
   - 处理节点从原群组移除的逻辑

10. removeNodeFromGroup: 从群组中移除节点
    - 删除节点的groupId属性
    - 从群组的nodeIds列表中移除节点ID

11. updateGroupBoundary: 根据群组内节点自动调整群组边界
    - 计算群组内所有节点的边界
    - 自动调整群组的位置和尺寸以包含所有节点
    - 添加内边距确保节点不会紧贴边界

**关键实现细节：**
- 深度合并机制：使用展开运算符确保更新时保留所有未明确指定的属性
- 类型安全：明确区分Node和Group类型，使用BlockEnum.GROUP进行类型判断
- 数据验证：在更新时验证必要字段（如title不能为空）

#### edgesSlice.ts
边状态管理切片，实现以下功能：

**状态属性：**
- edges: Edge[] - 存储所有边

**方法功能：**

1. addEdge: 添加新边
   - 将边添加到edges数组中
   - 包含source、target、label等基本属性

2. updateEdge: 更新边的特定属性
   - 实现部分更新，保留未修改的属性
   - 更新updatedAt时间戳

3. deleteEdge: 从状态中删除边
   - 从edges数组中过滤掉指定ID的边

4. getEdges & getEdgeById: 获取边的辅助方法

#### canvasViewSlice.ts
画布视图状态管理切片，实现以下功能：

**状态属性：**
- viewport: ViewportState - 视口状态，包含x, y, zoom
- canvasSize: { width: number; height: number } - 画布尺寸

**方法功能：**

1. setViewport: 设置整个视口状态
   - 替换当前视口位置和缩放

2. updateViewport: 部分更新视口状态
   - 只更新指定的视口属性

3. setCanvasSize: 设置画布尺寸

### 2. 图形组件模块 (components/graph/)

#### GraphPageContent.tsx
应用主入口组件，负责整合所有图形编辑功能：

**状态管理：**
- 使用useNodesState和useEdgesState管理ReactFlow状态
- 与useGraphStore同步图形数据

**渲染功能：**
- 节点类型：NoteNode（普通节点）和GroupNode（群组节点）
- 边类型：CustomEdge（自定义边）
- ReactFlow组件配置：缩放、连接模式、选择行为等

**事件处理：**
- onNodesChange: 核心状态同步逻辑

  - 处理节点删除
  - 处理尺寸调整（防抖机制）

- onConnect: 处理节点连接
- onNodeClick, onEdgeClick: 处理选择事件

**组件结构：**

**依赖的Hooks：**

## 核心技术机制

### 1. 状态同步机制
onNodesChange是ReactFlow与Zustand状态管理之间的关键桥梁，处理三种主要事件：

1. **位置变化**：拖拽节点后，在dragging: false事件中获取节点最终位置
2. **尺寸变化**：拖拽调整节点大小时，防抖处理以避免频繁更新
3. **选择/删除**：同步UI选择状态到store状态

### 2. 群组管理机制
- 群组内节点边界约束：节点不能被拖出群组边界
- 群组边界自动扩展：根据内部节点自动调整群组尺寸
- 群组移动同步：移动群组时同步移动内部所有节点

### 3. 性能优化策略
- React.memo避免不必要的重渲染
- useCallback优化回调函数
- 防抖处理频繁更新操作
- 深度合并避免状态丢失

## 设计原则和模式

1. **单一职责原则**
   - 节点组件只负责渲染
   - ReactFlow负责UI交互
   - onNodesChange是唯一的状态同步桥梁
   - Store只负责数据存储

2. **明确保留原则**
   - 更新节点时，明确保留所有未修改的属性
   - 使用深度合并确保不丢失任何属性

3. **防抖延迟原则**
   - 只在操作完成时更新store
   - 使用防抖避免频繁状态更新
   - UI先响应，状态后更新

## 项目特色功能

1. **节点类型**：支持普通节点和群组节点

3. **拖拽调整**：节点尺寸可拖拽调整
4. **位置约束**：节点在群组边界内的位置约束
5. **实时编辑**：节点标题双击编辑
